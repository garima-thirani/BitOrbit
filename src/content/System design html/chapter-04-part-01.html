Database Scaling Decisions

Database scaling is one of the hardest challenges in system design. Splitting a large database into smaller, independent databases called shards.

Database Sharding: Splitting a large database into smaller, independent databases called shards.
Sharding Strategies
Key-Based (Hash)
Shard key % N. Simple but adding shards requires rehashing.
Range-Based
Divide data by ranges. Simple but can cause hotspots.
Geographic
Shard by region. Good for data residency.
