The Supermarket Checkout

Imagine a supermarket with six checkout counters. If every customer joins Counter 1 while Counters 2-6 remain empty, the queue becomes enormous. Instead, a staff member directs each customer to an appropriate counter. That person is acting as a load balancer. The same idea applies to servers. Without a load balancer, thousands of users may accidentally hit the same server while others remain idle.

Formal Definition: A load balancer is a networking component that distributes incoming requests across multiple backend servers to improve scalability, availability, and fault tolerance.
If Server B fails, requests are automatically routed to the remaining healthy servers.
⚖ Layer 4 vs Layer 7

Layer 4 (Transport)

Works using IP addresses and TCP/UDP ports. Faster and lower latency. Use when application content is irrelevant. Examples: AWS NLB, HAProxy (TCP mode).

Layer 7 (Application)

Understands HTTP. Can inspect URL, headers, cookies, hostnames. Allows intelligent routing: /images → Image Service, /api → API Service. Examples: AWS ALB, NGINX, Envoy.

🔄 Load Balancing Algorithms
🔁 Round Robin
Requests rotate sequentially: 1→A, 2→B, 3→C, 4→A. Simple and fair if servers are equal. Ignores server load.
📊 Least Connections
Send traffic to the server with the fewest active connections. Better when requests have varying durations.
🔒 IP Hash
The client's IP determines the backend server. Useful for session affinity. Be careful: uneven IP distributions can create hotspots.
❤️ Health Checks

A load balancer continuously verifies backend health. Typical checks: HTTP 200 on /health, TCP connection, custom health endpoint. If a server fails, it is removed from rotation and traffic is redirected to healthy servers. This enables automatic failover.

🎯 Interview Callout
Q
"Why can't clients connect directly to servers?"
"A load balancer provides a single entry point, distributes requests, removes failed servers automatically, and enables horizontal scaling without clients knowing backend topology."
🚨 Common Mistakes
❌ Ignoring Health Checks
Without health checks, failed servers still receive traffic.
❌ Using Round Robin for Uneven Workloads
Some requests take longer. Least Connections is often better.
❌ Forgetting Stateless Services
Stateless services work best behind load balancers.
🧠 Memory Trick
One Door → Many Rooms. The load balancer is the front door. Users don't choose the room. The receptionist does.
