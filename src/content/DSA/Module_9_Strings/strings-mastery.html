🧠 The Story — The Word Puzzle
Real World Story — Search Engine Autocomplete

You type "appl" in Google. In milliseconds, it returns "apple", "application", "apply". How?

Under the hood, string algorithms power everything: autocomplete (Trie), text search (KMP/Rabin-Karp), spell check (edit distance), compression (LZ77), DNA alignment (LCS).

For interviews, most string problems fall into one of these patterns:
(1) Sliding window: "longest/shortest substring satisfying condition" → two pointers shrink/expand window.
(2) Two pointers: "palindrome check", "reverse words", "remove duplicates" → meet in middle or start-end crawl.
(3) Hash map: "frequency check", "anagram check", "character mapping".
(4) DP: "edit distance", "palindromic subsequence", "word break".

Strings feel different from arrays but they're really the same memory. The trick that unlocks almost every string problem is the sliding window: maintain two pointers l and r around a "window" that moves right. Whenever the window is invalid, shrink it from the left; whenever it's valid, record your best answer. This converts O(n²) brute-force into O(n) because the left pointer never moves left. For pattern matching, KMP does the same thing to pattern comparison: a precomputed jump table means you never re-examine a character you've already matched.

🪟 When to use sliding window

  • "Longest/shortest substring satisfying X"
  • "All substrings with exactly k distinct chars"
  • Window condition is monotone: shrinking the window can only make it more valid, not less

↔️ The two-pointer invariant

l only moves right. r only moves right. So the total iterations are at most 2n. Every character enters the window once and leaves once.

🔢 KMP — LPS as a jump table

On mismatch at pattern[j], jump to lps[j−1] — the longest prefix that's also a suffix of what matched so far. The text pointer never retreats; the pattern pointer never goes past what's necessary.

🔤 Hash map for frequency

Track how many required characters the window currently satisfies with a have counter. Only when have == required is the window valid. Increment/decrement have only when a frequency crosses the threshold.

🌍 Where this lives in the real world
  • Browser find (Ctrl+F): KMP or Boyer-Moore under the hood — highlight all occurrences in one linear scan.
  • grep / awk: pattern matching against millions of lines is KMP or Aho-Corasick (multi-pattern KMP).
  • Autocomplete / Trie: prefix sliding through a trie; each character narrows the candidate set.
  • DNA / protein alignment: "find all positions where this 20-base primer appears" is KMP on a genome.
  • Rate limiting & analytics: "requests in the last 60 seconds" is a sliding window over a timestamp array.
  • Spam filter: longest common subsequence between a message and known templates — string DP.
⚠️ The traps

Shrinking when you shouldn't: only shrink when the window is invalid and you've already recorded the current best. If you shrink too eagerly you'll miss the answer. Frequency threshold logic: for min-window substring, increment have only when the window count equals (not exceeds) the needed count — over-counting a character doesn't add a new satisfied type. KMP lps off-by-one: when mismatch and j > 0, set j = lps[j−1], not j -= 1; the jump can be much larger than 1.

🎬 Minimum Window Substring — Sliding Window Mastery
🔍
Pattern: Expand right until window is valid (contains all required chars with required freq). Then shrink from left to find minimum valid window. Track result when valid.
s="ADOBECODEBANC", t="ABC" need={A:1,B:1,C:1}, have=0, formed=0 Expand r until we have all of t: window "ADOBEC" → have all 3 → formed=3 Shrink l: "DOBEC" → lose A → not valid Expand r again... "DOBECODEBA" → A back, but need min Eventually: "BANC" (length 4) → minimum window ✓
Python · Minimum Window Substring — O(n+m)
from collections import Counter

def min_window(s, t):
    need = Counter(t)                    # required character frequencies
    have, required = 0, len(need)        # formed char types vs total needed types
    window = {}
    best = ""
    l = 0

    for r, ch in enumerate(s):
        window[ch] = window.get(ch, 0) + 1          # expand: add s[r] to window
        if ch in need and window[ch] == need[ch]:
            have += 1                               # one more char type satisfied

        while have == required:                    # valid window: try shrinking
            if not best or r-l+1 < len(best):
                best = s[l:r+1]                    # update minimum
            lch = s[l]; window[lch] -= 1
            if lch in need and window[lch] < need[lch]:
                have -= 1                          # lost a required char type
            l += 1                                 # shrink

    return best
🎬 Sliding window — expand right until valid, shrink left until invalid, record best
Right never stops; left only chases. Whenever the window holds all required characters, record the length and push l one step right to see if you can do better. The window "breathes": expand until valid, shrink until invalid, expand again. Each character enters and leaves exactly once — O(n). ▶ loops every 12s.

🎬 KMP — Linear Time Pattern Matching
The KMP Story

Naive string search: for each position in text, compare all m characters of pattern. O(n*m) worst case.

KMP insight: When a mismatch occurs after k successful matches, we don't need to restart from scratch. The pattern itself tells us the longest prefix that is also a suffix of the matched part — we can jump to that position.

The failure function (LPS array) precomputes for each position i in pattern: the length of the longest proper prefix of pattern[0..i] that is also a suffix. This is computed in O(m). Then matching takes O(n). Total: O(n+m).

Python · KMP Pattern Matching — O(n+m)
def kmp_search(text, pattern):
    n, m = len(text), len(pattern)
    if m == 0: return [0]

    # Build LPS (Longest Proper Prefix = Suffix) array
    lps = [0] * m
    length, i = 0, 1
    while i < m:
        if pattern[i] == pattern[length]:
            length += 1; lps[i] = length; i += 1
        elif length:
            length = lps[length - 1]    # don't increment i, try shorter prefix
        else:
            lps[i] = 0; i += 1

    # Match using LPS
    matches, i, j = [], 0, 0
    while i < n:
        if text[i] == pattern[j]: i += 1; j += 1
        if j == m:
            matches.append(i - j)       # found match at position i-j
            j = lps[j - 1]              # look for more matches (overlap possible)
        elif i < n and text[i] != pattern[j]:
            if j: j = lps[j - 1]      # use LPS to skip redundant comparisons
            else: i += 1              # no prefix match, advance text
    return matches
🎬 KMP — the LPS array lets you skip redundant comparisons
Never compare a character you've already matched. The LPS array says: "if you fail here, the first lps[j−1] characters of the pattern are already confirmed to match — skip straight there." Naive search backtracks i; KMP never backtracks i. ▶ loops every 12s.

🎬 String DP Problems
Python · Palindrome, Anagram, Valid IP
# --- Longest Palindromic Substring (Expand Around Center) ---
def longest_palindrome(s):
    res, best = "", 0
    for i in range(len(s)):
        for l, r in [(i,i), (i,i+1)]:   # odd and even centers
            while l >= 0 and r < len(s) and s[l] == s[r]:
                if r-l+1 > best: best = r-l+1; res = s[l:r+1]
                l -= 1; r += 1
    return res

# --- Valid Anagram ---
def is_anagram(s, t):
    return Counter(s) == Counter(t)   # O(n) time and space

# --- Reverse Words in String ---
def reverse_words(s):
    return ' '.join(s.split()[::-1])  # split handles multiple spaces

# --- Check if string rotation of another ---
# "abcde" rotated → "cdeab". Key: s2 is always a substring of s1+s1.
def is_rotation(s1, s2):
    return len(s1)==len(s2) and s2 in s1+s1

Longest Substring Without Repeating Characters
Problem (LeetCode 3): Given a string, find the length of the longest substring without repeating characters. E.g. 'abcabcbb' → 3 ('abc').
Sliding window + hash map: Track the last seen index of each character. When a duplicate is found at index r, jump left pointer directly to max(left, last_seen[char]+1). This skips all redundant shrinks and makes the window always valid.
🔍 Step-by-Step Trace
s = "abcabcbb" l=0 last_seen={} max_len=0 r=0 'a': last_seen={a:0} window=[a] len=1 r=1 'b': last_seen={a:0,b:1} window=[ab] len=2 r=2 'c': last_seen={a:0,b:1,c:2} len=3 r=3 'a': 'a' seen at 0, l=max(0,0+1)=1 window=[bca] last_seen[a]=3 len=3 r=4 'b': 'b' seen at 1, l=max(1,1+1)=2 window=[cab] last_seen[b]=4 len=3 r=5 'c': 'c' seen at 2, l=max(2,2+1)=3 window=[abc] last_seen[c]=5 len=3 r=6 'b': 'b' seen at 4, l=max(3,4+1)=5 window=[cb] len=2 r=7 'b': 'b' seen at 6, l=max(5,6+1)=7 window=[b] len=1 max_len = 3 ✓
💻 Solution
Python · Longest Substring Without Repeating
def length_of_longest_substring(s):
    last_seen = {}
    l, max_len = 0, 0
    for r, ch in enumerate(s):
        if ch in last_seen and last_seen[ch] >= l:
            l = last_seen[ch] + 1      # jump past the duplicate
        last_seen[ch] = r
        max_len = max(max_len, r - l + 1)
    return max_len
⌛ Complexity
Time: O(n) Space: O(min(n, charset))

Find All Anagrams in a String
Problem (LeetCode 438): Given strings s and p, return start indices of all p's anagrams in s. An anagram of p is any permutation of p's characters.
Fixed-size sliding window with frequency counter: Maintain a window of size len(p). Track how many characters currently have matching frequency ('matches' counter). Window is valid when matches==26 (all 26 letters balanced). Slide right by adding new char and removing old char.
🔍 Step-by-Step Trace

No. (only 3 chars so matches can be 26 only with full 26) Better: track only chars in p. matches=len(p)=3 if all match. Use p_count, w_count, have/need: have=0, need=3 (distinct chars in p with correct freq) Window [c,b,a]: have=3, need=3 → record index 0 Slide to [b,a,e]: remove c, add e → have=2 (c gone) ...skip... Window [e,b,a]: have=2 → nope Window [b,a,b]: have=2 (b freq=2 > p_count[b]=1) → nope Window [a,b,a]: have=1 → nope Window [b,a,c]: have=3 → record index 6 ✓ Result: [0, 6]
💻 Solution
Python · Find All Anagrams
from collections import Counter
def find_anagrams(s, p):
    p_count = Counter(p)
    w_count = Counter()
    result = []
    have, need = 0, len(p_count)   # distinct chars needing correct freq
    l = 0

    for r, ch in enumerate(s):
        w_count[ch] += 1
        if ch in p_count and w_count[ch] == p_count[ch]:
            have += 1

        if r - l + 1 > len(p):      # window too large, shrink
            lc = s[l]; l += 1
            if lc in p_count and w_count[lc] == p_count[lc]:
                have -= 1
            w_count[lc] -= 1

        if have == need:
            result.append(l)
    return result
⌛ Complexity
Time: O(n) Space: O(1) — 26 char alphabet

Word Break II — All Sentences
Problem (LeetCode 140): Given string s and dictionary wordDict, return all ways to add spaces to form a sentence using only dictionary words. E.g. s='catsanddog' → ['cats and dog', 'cat sand dog'].
DFS + memoization: Recursively try every word that starts at current index. If the remaining string s[i:] was already computed, return cached result. Without memo: exponential repeated work. With memo: polynomial since each suffix computed once.
🔍 Step-by-Step Trace
s="catsanddog" dict=["cat","cats","and","sand","dog"] dfs(0) — from index 0: "cat" matches → dfs(3) = ["sand dog", "and dog"] from index 3 "sand" matches → dfs(7) = ["dog"] "dog" matches → dfs(10) = [""] → return ["dog"] "and" matches → dfs(6) = ["dog"] "dog" matches → return ["dog"] → dfs(3) cached = ["sand dog", "and dog"] prepend "cat " → ["cat sand dog", "cat and dog"] "cats" matches → dfs(4) = ["and dog"] from index 4 "and" matches → dfs(7) = ["dog"] (cached!) → dfs(4) = ["and dog"] prepend "cats " → ["cats and dog"] dfs(0) = ["cat sand dog","cat and dog","cats and dog"] ✓
💻 Solution
Python · Word Break II (DFS + Memo)
def word_break(s, wordDict):
    word_set = set(wordDict)
    memo = {}

    def dfs(start):
        if start in memo: return memo[start]
        if start == len(s): return [""]   # base: end of string
        result = []
        for end in range(start+1, len(s)+1):
            word = s[start:end]
            if word in word_set:
                for rest in dfs(end):
                    result.append(word + (" " + rest if rest else ""))
        memo[start] = result
        return result
    return dfs(0)
⌛ Complexity
Time: O(n²×2^n) worst Space: O(2^n) for all sentences
