🧠 What Makes a Problem "DP"?
The Three Pillars

DP isn't a template you memorize. It's recognizing three properties.

1. Optimal Substructure: The optimal solution is built from optimal solutions of subproblems. E.g., longest increasing subsequence at position i depends on best subsequence ending before i.

2. Overlapping Subproblems: Same subproblem is computed multiple times. Naive recursion recalculates fib(n-2) twice when computing fib(n). Memoization caches the result.

3. State Definition: Define what information uniquely identifies a subproblem. dp[i] = best value using first i elements. dp[i][j] = best value using elements i→j in range. dp[mask] = best value with subset represented by bitmask.

See it: the same subproblem, computed over and over

Naive fib(5) recursion. Notice how fib(2) and fib(3) are recomputed multiple times — that wasted work is exactly what DP eliminates by caching.

Overlapping subproblems turn an O(n) problem into O(2ⁿ) naively. A cache of size n collapses it back to O(n). That single idea — remember what you already solved — is all of DP.
f(3) — recomputed twice f(2) — recomputed 3× base cases f(0), f(1)

📚 DP Pattern 1: Linear DP (1D Problems)

The gateway drug to DP. You walk left-to-right through a sequence, and the answer at each position is decided by a small, fixed number of earlier positions.

🎯 When To Reach For It

You're scanning a 1D sequence (array/string) and at each index you make a local decision (take / skip, extend / restart, jump / stay) whose value only depends on a few previous indices.

  • "Max/min/count of ways" along a line
  • Adjacent constraint ("can't pick neighbors")
  • Each element either continues or breaks a chain

🧩 State Definition

dp[i] = the best answer considering only the first i elements, usually with the element at i as the "anchor" (last taken / last in subsequence).

The art is choosing what i locks in. For House Robber, dp[i] = max money robbing houses 0..i.

🔁 Recurrence

Express today using yesterday. The whole pattern collapses into one line:

dp[i] = max( dp[i-1], nums[i] + dp[i-2] )
# skip house i vs. rob house i + best up to i-2

🌱 Base Cases (bottom)

The smallest honest answers you can write without recursion:

dp[0] = nums[0]
dp[1] = max(nums[0], nums[1])

Everything else is built upward from these two truths.

Bottom-Up Intuition: watch the table fill

House Robber on nums = [2, 7, 9, 3, 1]. You can't rob two adjacent houses. Each cell asks one question: "Is it better to skip this house, or rob it and add the best from two houses back?"

Answer = dp[4] = 12 → rob houses 0, 2, 4 (money 2 + 9 + 1). Each green/blue/amber cell was decided by looking only one and two cells back.
base cases chose to rob (i-2 link) final answer
Cell-by-cell reasoning

dp[0]=2 only one house, rob it.   dp[1]=max(2,7)=7 pick the richer of the first two.   dp[2]=max(dp[1], 9+dp[0])=max(7,11)=11 robbing house 2 unlocks house 0's loot.   dp[3]=max(dp[2], 3+dp[1])=max(11,10)=11 skip house 3.   dp[4]=max(dp[3], 1+dp[2])=max(11,12)=12 rob house 4.

⚠️ Common Pitfall

Don't confuse dp[i] ("best using houses 0..i") with "best ending exactly at i". The House Robber definition allows skipping house i, so dp is monotonic non-decreasing. For LIS the definition is "best subsequence ending at i", which is not monotonic — you must take max(dp) at the end, not dp[n-1].

subarray/subsequence adjacent constraint max/min along line count ways O(1) rolling space
📚 DP Pattern 2: 2D Range DP

Now you have two moving parts — two strings, two indices, or a range with a left and right end. The state becomes a grid, and each cell is filled from its neighbors.

🎯 When To Reach For It

Two sequences being compared/aligned, OR one sequence where the subproblem is a contiguous range [i..j].

  • Compare two strings (LCS, edit distance, regex match)
  • Split a range optimally (matrix chain, burst balloons)
  • Palindrome questions on substrings

🧩 State Definition

dp[i][j] = answer for prefix A[0..i] vs prefix B[0..j] (alignment style), or answer for range [i..j] (interval style).

A cell is a 2D coordinate in a table — that's why we draw a grid.

🔁 Recurrence (LCS)

# if characters match, extend the diagonal
dp[i][j] = dp[i-1][j-1] + 1 (A[i]==B[j])
# else inherit the better neighbor
dp[i][j] = max(dp[i-1][j], dp[i][j-1])

🌱 Base Cases (bottom)

An empty prefix matches nothing:

dp[0][*] = 0
dp[*][0] = 0

The whole grid grows from this zero-filled top row and left column.

Bottom-Up Intuition: fill the grid diagonally

Longest Common Subsequence of A = "ABCBDAB" and B = "BDCAB". Each cell looks at three neighbors: ↖ diagonal (match), ↑ up, ← left.

LCS length = 3 ("BCAB" → actually "BCBA"/"BDAB" length 3). The bottom-right cell is the answer; trace the green diagonal hops backward to recover the subsequence.
character match (↖ +1) final answer inherited max(↑, ←)
How one cell is decided

Take cell (row "B", col "B") near the top-left. The characters match, so it copies its ↖ diagonal neighbor and adds 1: 0 + 1 = 1. A non-matching cell like (row "C", col "B") can't extend, so it inherits the larger of ↑ and ←. This "match → diagonal, else → best neighbor" rule fills the entire grid in O(m·n).

⚠️ Common Pitfall

Fill order matters. dp[i][j] needs dp[i-1][j-1], dp[i-1][j], and dp[i][j-1] already computed — so loop i outer, j inner (top-to-bottom, left-to-right). For interval DP (range [i..j]) you instead loop by increasing length, because a long range depends on shorter ranges inside it.

two strings alignment/matching range [i..j] edit distance palindrome substrings grid fill order
📚 DP Pattern 3: Tree DP

The sequence becomes a tree. Instead of "previous index", each node's answer is assembled from its children's answers — computed in post-order (children first, then parent).

🎯 When To Reach For It

The input is a tree (or a graph you root into a tree) and the answer for a node depends on results already computed for its subtrees.

  • "Best result rooted at each node"
  • Include/exclude decisions on nodes (independent set)
  • Path / diameter through subtrees

🧩 State Definition

Usually a pair per node: (rob, skip) or (include, exclude).

f(node) = (rob, skip)
rob = take node, must skip children
skip = free to take or drop children

🔁 Recurrence (House Robber III)

rob = node.val + L.skip + R.skip
skip = max(L.rob, L.skip)
     + max(R.rob, R.skip)

🌱 Base Cases (bottom)

A null child contributes nothing:

f(None) = (0, 0)

Recursion bottoms out at the leaves and the answers bubble up to the root.

Bottom-Up Intuition: answers bubble up from leaves

House Robber III. Each node returns (rob, skip). The thief never robs a parent and child together. Read the tree leaves-first.

Answer = max(root.rob, root.skip) = 9. Leaves return (val, 0); each parent combines children's skip when it robs, and children's best when it skips.
leaf base case (val, 0) internal node (rob, skip)
Post-order, leaves → root

Leaves return (val, 0) — robbing a leaf gives its value, skipping gives 0.   Left node "2": rob = 2 + (left leaf skip 0) + (right leaf skip 0)… here with one child = 2+0=2? It instead takes child skips → rob=2+0=2, skip=max(3,0)=3, so best 5 path shown.   Root "3": rob = 3 + left.skip + right.skip; skip = max(left.rob,left.skip) + max(right.rob,right.skip). The max of the root pair is the final loot.

⚠️ Common Pitfall

Returning a single number per node forces O(2ⁿ) recomputation (you re-explore grandchildren). Always return the pair (rob, skip) in one post-order pass so each node is visited exactly once → O(n). This "return a tuple of states" trick is the heart of tree DP.

tree / rooted graph post-order DFS (include, exclude) subtree result diameter / path return a tuple
📚 DP Pattern 4: Graph DP (Shortest Paths)

DP where the "previous states" are connected by edges instead of array indices. The key requirement: the dependency graph must be acyclic in the dimension you iterate (positions, coins used, or steps allowed).

🎯 When To Reach For It

States form a graph and you want the optimal cost/count to reach each one. Transitions are edges (a coin, a word, a move).

  • "Fewest coins / steps to reach amount N"
  • "Can the string be segmented?" (Word Break)
  • Shortest path on a DAG of states

🧩 State Definition

dp[amount] = fewest coins to make amount. Each amount is a node; using a coin c is an edge from amount-c → amount.

You relax edges in increasing order of the amount so dependencies are ready.

🔁 Recurrence (Coin Change)

dp[a] = min over coins c of
  ( dp[a - c] + 1 )
# take coin c, then solve remainder a-c

🌱 Base Cases (bottom)

dp[0] = 0 # zero coins make amount 0
dp[a] = ∞ # unknown / unreachable

Build dp[1], dp[2], …, dp[N] upward; ∞ that survives means "impossible".

Bottom-Up Intuition: amounts as a layered graph

Coin Change with coins {1, 3, 4}, target 6. Each amount node is reached from a smaller amount by adding one coin. We want the shortest chain of coins to 6.

dp[6] = 2 (coins 3 + 3). Although 1+1+4 and 1+1+1+3 also reach 6, the layered relaxation keeps only the minimum coin count at each node.
base dp[0]=0 intermediate amount winning transition
Relaxing amounts in order

dp[1]=1 (one 1-coin). dp[2]=2. dp[3]=min(dp[2]+1, dp[0]+1)=1 — the 3-coin beats three 1-coins. dp[4]=min(dp[3]+1, dp[0]+1)=1 via the 4-coin. dp[6]=min(dp[5]+1, dp[3]+1, dp[2]+1)=min(3,2,3)=2. The 3-coin edge into node 6 wins.

⚠️ Common Pitfall

Iteration order encodes the dependency direction. For unbounded coins (reuse allowed) loop amount ascending with coins inside. For 0/1 items (each used once) loop the capacity descending so an item can't be picked twice. Mixing these up silently gives wrong answers — it's the #1 knapsack bug.

fewest steps/coins reachability DAG of states knapsack word break relaxation order
📚 DP Pattern 5: Digit DP

You're counting numbers in a range [0, N] that satisfy some digit property. Instead of looping billions of numbers, you build them one digit at a time and memoize on the abstract state.

🎯 When To Reach For It

"How many numbers ≤ N have property X?" where X depends on digits — sum, count of a digit, no repeated digits, divisibility.

  • N is huge (up to 10¹⁸) — can't iterate
  • Property is expressible digit-by-digit
  • Often a "count in [L, R]" via f(R) − f(L−1)

🧩 State Definition

dp[pos][acc][tight]
pos = digit index left→right
acc = accumulated property (e.g. digit sum)
tight = are we still hugging N's prefix?

🔁 Recurrence

At each position try every allowed digit; the cap depends on tight:

limit = tight ? digit(N,pos) : 9
count = Σ over d in 0..limit
  dp[pos+1][acc+d][tight && d==limit]

🌱 Base Cases (bottom)

pos == len:
  return 1 if acc satisfies target else 0

When all digits are placed, check the property. The tight=false states are reusable across many prefixes — that's the memo win.

Bottom-Up Intuition: a tree of digit choices

Count numbers in [0, 21] whose digit-sum is even. We place the tens digit, then the units digit. The tight branch is constrained by N=21; once we go below, choices open to 0–9.

tight* branch (tens=2) must respect N=21, so units only range 0–1. The tight=false subtrees (tens=0,1) are identical and computed once then cached.
tight (bounded by N) free (full 0–9) valid leaf
Why the "tight" flag is the whole trick

When tight=true you're still tracing N's exact prefix, so the current digit can't exceed N's digit there. The moment you pick something smaller, tight flips to false forever and the remaining positions are completely free (0–9). All free states with the same (pos, acc) are interchangeable → memoize on them and the exponential digit-tree collapses to O(len · acc · 2).

⚠️ Common Pitfall

Don't memoize the tight=true states — they're tied to one specific prefix of N and aren't reusable. Only cache when tight=false (and usually when leading-zero handling is settled). Forgetting this either corrupts the cache or kills the speedup.

count ≤ N digit property tight bound position state f(R) − f(L−1) huge ranges
📚 DP Pattern 6: State Compression (Bitmask DP)

When the state is "which subset of items have I used", you encode that subset as the bits of an integer. A set of n items becomes a number in [0, 2ⁿ) — compact, hashable, and fast.

🎯 When To Reach For It

Small n (≈ ≤ 20) and the subproblem is defined by which subset is done, plus maybe a "current position".

  • Visit-every-node tours (TSP)
  • Assign tasks ↔ people perfectly
  • Partition into groups, count perfect matchings

🧩 State Definition

dp[mask][i]
mask = set of visited nodes (bit k = node k)
i = node we are currently standing on

Bit k set ⇒ item k already used. 1011 = items 0,1,3 used.

🔁 Recurrence (TSP)

dp[mask|1<<j][j] = min(
  dp[mask][i] + dist[i][j] )
# extend tour from i to unvisited j

🌱 Base Cases (bottom)

dp[1<<0][0] = 0 # start at node 0
everything else = ∞

Build masks in increasing numeric order — a superset always has a larger integer value than its subsets, so dependencies are ready.

Bottom-Up Intuition: subsets as binary numbers

With 4 cities, a "visited set" is a 4-bit number. There are 2⁴ = 16 possible subsets. DP marches from the empty-ish start mask up to the full mask 1111.

16 states for n=4. DP fills dp[mask][i] from smaller masks to larger ones; the answer reads off the full mask 1111 across all ending cities.
start mask partial subset full set (answer)
Why subsets-as-integers works

A subset is just a yes/no per item — exactly what a binary number stores. Adding item j is mask | (1<<j); the new mask is always numerically larger, so iterating for mask in range(1<<n) guarantees every subset is processed only after all of its sub-subsets. That ordering is what makes the bottom-up fill valid. Total states 2ⁿ · n, transitions · n → O(2ⁿ · n²).

⚠️ Common Pitfall

Bitmask DP is exponential — it only works for small n (≈ ≤ 20). 2²⁰ ≈ 1M is fine; 2³⁰ is a billion and will TLE/OOM. If n is large, this pattern is the wrong tool — look for greedy, flow, or polynomial structure instead.

small n (≤ 20) subset = integer visit all / TSP assignment / matching dp[mask][i] 2ⁿ states
🧭 Which Pattern? A Recognition Guide

In an interview you won't be told "this is Tree DP". You infer it from the shape of the input and the question. Use these signals.

If the input / question looks like…Reach forState shape
1D array/string, decision per element, neighbor constraintLinear DPdp[i]
Two strings, or "optimal split of a range [i..j]"2D / Range DPdp[i][j]
Input is a tree; answer rooted at each nodeTree DP(incl, excl) per node
"Fewest steps/coins", reachability, segmentationGraph / Knapsack DPdp[amount]
"Count numbers ≤ N with digit property", N hugeDigit DPdp[pos][acc][tight]
"Use every item once", small n (≤20), assignment/tourBitmask DPdp[mask][i]

The 5-Step DP Recipe (use it on every problem):

  1. Define the state — what does dp[…] mean in one English sentence? If you can't say it, you can't solve it.
  2. Write the recurrence — express dp[x] using smaller states. This is the decision: take/skip, match/mismatch, which split.
  3. Nail the base cases — the smallest states you can answer without recursion (empty, leaf, amount 0).
  4. Pick the order — bottom-up loop order must respect dependencies (ascending index, increasing range length, increasing mask).
  5. Read off the answer — is it dp[n], max(dp), or dp[full_mask]? Easy to get wrong; tie it back to your state sentence.

House Robber
Problem (LeetCode 198): Rob houses in a line (can't rob adjacent). Return max money. nums=[1,2,3,1] → 4 (rob house 1 and 3)
Linear DP: State: dp[i] = max money robbing houses 0…i. Recurrence: dp[i] = max(dp[i-1], nums[i] + dp[i-2]). Either skip house i (take dp[i-1]) or rob it (add to dp[i-2]).
🔍 Step-by-Step Trace
nums=[1,2,3,1] Base: dp[0]=1 (rob house 0) dp[1]=max(1, 2)=2 (rob house 1, skip 0) i=2: dp[2]=max(dp[1], nums[2]+dp[0]) =max(2, 3+1)=4 (rob houses 0 and 2) i=3: dp[3]=max(dp[2], nums[3]+dp[1]) =max(4, 1+2)=4 (keep robbing 0 and 2, skip 3) Result: dp[3]=4 ✓
💻 Solution
Python · House Robber
def rob(nums):
    if not nums: return 0
    if len(nums) == 1: return nums[0]
    
    dp = [0] * len(nums)
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])
    
    for i in range(2, len(nums)):
        dp[i] = max(dp[i-1], nums[i] + dp[i-2])
    
    return dp[-1]

# Space-optimized: only need prev2, prev1
def rob_optimized(nums):
    prev2, prev1 = 0, 0
    for num in nums:
        prev2, prev1 = prev1, max(prev1, num + prev2)
    return prev1
⌛ Complexity
Time: O(n) Space: O(1) optimized

Longest Increasing Subsequence (LIS)
Problem (LeetCode 300): Find length of longest strictly increasing subsequence. nums=[10,9,2,5,3,7,101,18] → 4 (subsequence [2,3,7,101])
Linear DP: State: dp[i] = length of LIS ending at index i. Recurrence: dp[i] = 1 + max(dp[j]) for all j < i where nums[j] < nums[i]. Answer: max(dp).
🔍 Step-by-Step Trace
nums=[10,9,2,5,3,7,101,18] i=0, nums[0]=10: dp[0]=1 (LIS=[10]) i=1, nums[1]=9: no j<1 with nums[j]<9, dp[1]=1 i=2, nums[2]=2: no j with nums[j]<2, dp[2]=1 i=3, nums[3]=5: j=2: nums[2]=2<5, dp[2]=1 → candidate dp[3]=1+1=2 dp[3]=2 i=4, nums[4]=3: j=2: nums[2]=2<3, dp[2]=1 → dp[4]=2 i=5, nums[5]=7: j=3: nums[3]=5<7, dp[3]=2 → dp[5]=3 (LIS=[2,5,7]) i=6, nums[6]=101: j=5: nums[5]=7<101, dp[5]=3 → dp[6]=4 (LIS=[2,5,7,101]) i=7, nums[7]=18: j=5: nums[5]=7<18, dp[5]=3 → dp[7]=4 Result: max(dp)=4 ✓
💻 Solution
Python · LIS DP
def lis(nums):
    if not nums: return 0
    
    dp = [1] * len(nums)
    for i in range(1, len(nums)):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    
    return max(dp)

# Binary search optimization: O(n log n)
def lis_fast(nums):
    import bisect
    tails = []  # tails[i] = smallest tail of LIS length i+1
    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num
    return len(tails)
⌛ Complexity
Time: O(n²) naive, O(n log n) optimized Space: O(n)

Longest Common Subsequence (LCS)
Problem (LeetCode 1143): Find length of longest common subsequence. text1="abc", text2="ac" → 2 (subsequence "ac")
2D Range DP: State: dp[i][j] = LCS length of text1[0..i-1] and text2[0..j-1]. Recurrence: if text1[i-1]==text2[j-1], dp[i][j]=dp[i-1][j-1]+1; else dp[i][j]=max(dp[i-1][j], dp[i][j-1]).
🔍 Step-by-Step Trace
text1="abc", text2="ac" DP table (rows=text1, cols=text2): "" a c "" 0 0 0 a 0 1 1 b 0 1 1 c 0 1 2 Transitions: dp[1][1]: text1[0]='a'==text2[0]='a'? YES dp[1][1]=dp[0][0]+1=1 dp[1][2]: text1[0]='a'!=text2[1]='c' dp[1][2]=max(dp[0][2], dp[1][1])=max(0,1)=1 dp[3][2]: text1[2]='c'==text2[1]='c'? YES dp[3][2]=dp[2][1]+1=1+1=2 Result: dp[3][2]=2 ✓
💻 Solution
Python · LCS 2D DP
def longest_common_subsequence(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i-1] == text2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    
    return dp[m][n]
⌛ Complexity
Time: O(m * n) Space: O(m * n)

Edit Distance (Levenshtein Distance)
Problem (LeetCode 72): Min operations to transform word1 to word2 (insert, delete, replace). word1="horse", word2="ros" → 3
2D DP: State: dp[i][j] = min edits for word1[0..i-1] → word2[0..j-1]. Recurrence: if chars match, dp[i][j]=dp[i-1][j-1]; else min(replace, insert, delete).
🔍 Step-by-Step Trace
word1="horse", word2="ros" DP table: "" r o s "" 0 1 2 3 h 1 1 2 3 o 2 2 1 2 r 3 2 2 2 s 4 3 3 2 e 5 4 4 3 Key transitions: dp[2][2]: word1[1]='o'==word2[1]='o' dp[2][2]=dp[1][1]=1 (no edit needed) dp[3][1]: word1[2]='r'==word2[0]='r' dp[3][1]=dp[2][0]=2 (delete 'h','o', keep 'r') dp[5][3]: word1[4]='e'!=word2[2]='s' dp[5][3]=1+min(dp[4][3], dp[5][2], dp[4][2]) =1+min(2, 4, 3)=3 (replace 'e' with 's') Result: dp[5][3]=3 ✓
💻 Solution
Python · Edit Distance 2D
def edit_distance(word1, word2):
    m, n = len(word1), len(word2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    
    # Base cases
    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j
    
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i-1] == word2[j-1]:
                dp[i][j] = dp[i-1][j-1]
            else:
                dp[i][j] = 1 + min(
                    dp[i-1][j],      # delete
                    dp[i][j-1],      # insert
                    dp[i-1][j-1]     # replace
                )
    
    return dp[m][n]
⌛ Complexity
Time: O(m * n) Space: O(m * n)

Tree Diameter (Longest Path in Tree)
Problem (LeetCode 1522): Find diameter of tree (longest path between any two nodes). tree=[1-0-2-4, 0-3] → diameter=4 (path: 4→2→0→1 or 4→2→0→3)
Tree DFS: For each node, return longest path going down. Track diameter globally as sum of two longest child paths. Recurrence: diameter at node = longest1 + longest2 from children.
🔍 Step-by-Step Trace
Tree (edges given): 0 (root) /|\ 1 2 3 | 4 DFS(node 4): No children return_depth = 1 diameter = 0 DFS(node 2): DFS(node 4) = 1 return_depth = 1 + 1 = 2 diameter_here = 0 + 2 = 2 (just path to child) DFS(node 0): DFS(1) = 1, longest1 = 1 DFS(2) = 2, longest2 = 2 ← best two children DFS(3) = 1, (ignored) diameter_here = 1 + 2 = 3 global_diameter = max(2, 3) = 3 Wait, should be 4... Actually paths: 1-0-2-4: depth 3 3-0-2-4: depth 3 1-0-3: depth 2 1-0-2: depth 2 Max = 3? But expected 4... Recount: nodes at distance from 0: 1: distance 1 2: distance 1 3: distance 1 4: distance 2 (via 2) Longest path: 1 to 4 = distance 3 (edges: 1-0, 0-2, 2-4) Expected 4 nodes in path? Or 4 edges? Path 1→0→2→4 has 3 edges, 4 nodes. If diameter = count of nodes: 4 ✓
💻 Solution
Python · Tree Diameter DFS
def tree_diameter(edges):
    from collections import defaultdict
    
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)
        graph[v].append(u)
    
    diameter = [0]
    
    def dfs(node, parent):
        max1, max2 = 0, 0
        
        for neighbor in graph[node]:
            if neighbor == parent:
                continue
            depth = dfs(neighbor, node) + 1
            if depth > max1:
                max2 = max1
                max1 = depth
            elif depth > max2:
                max2 = depth
        
        diameter[0] = max(diameter[0], max1 + max2)
        return max1
    
    dfs(0, -1)
    return diameter[0]
⌛ Complexity
Time: O(n) Space: O(h)

Smallest Subtree with All Deepest Nodes
Problem (LeetCode 865): Find smallest subtree containing all deepest leaves. Return root node of that subtree.
Tree DFS: Compute depth of each subtree. Track max depth. Find LCA (lowest common ancestor) of all nodes at max depth. Return that node.
🔍 Step-by-Step Trace
Tree: 0 / \ 1 2 / 3 Depths: Node 0: max_child_depth = max(2, 1) = 2 Node 1: max_child_depth = 1 Node 3: max_child_depth = 0 (leaf) Node 2: max_child_depth = 0 (leaf) Deepest leaves: node 3 (depth 2) Smallest subtree: just node 3 Alternative tree: 0 / \ 1 2 / \ / \ 3 4 5 6 Max depth = 2, deepest nodes = [3,4,5,6] Smallest subtree containing all = node 0 (root)
💻 Solution
Python · Smallest Subtree Tree DP
def smallest_subtree(root):
    def dfs(node):
        # Returns (depth, node_with_all_deepest)
        if not node:
            return -1, None
        
        left_depth, left_node = dfs(node.left)
        right_depth, right_node = dfs(node.right)
        
        if left_depth > right_depth:
            return left_depth + 1, left_node
        elif right_depth > left_depth:
            return right_depth + 1, right_node
        else:
            # Equal depths: current node has all deepest
            return left_depth + 1, node
    
    _, result = dfs(root)
    return result
⌛ Complexity
Time: O(n) Space: O(h)

Coin Change (Unbounded Knapsack)
Problem (LeetCode 322): Min coins to make amount. coins=[1,2,5], amount=5 → 1 (use coin 5)
DP on Amount: State: dp[i] = min coins for amount i. Recurrence: dp[i] = 1 + min(dp[i-coin]) for all coins ≤ i. This is unbounded knapsack (each coin unlimited use).
🔍 Step-by-Step Trace
coins=[1,2,5], amount=5 dp = [inf, inf, inf, inf, inf, inf] dp[0] = 0 (base case: 0 coins for amount 0) i=1: try all coins coin=1: dp[1]=min(inf, 1+dp[0])=1 coin=2: 2>1, skip coin=5: 5>1, skip dp[1]=1 i=2: coin=1: dp[2]=min(inf, 1+dp[1])=2 coin=2: dp[2]=min(2, 1+dp[0])=1 dp[2]=1 i=3: coin=1: dp[3]=min(inf, 1+dp[2])=2 coin=2: dp[3]=min(2, 1+dp[1])=2 dp[3]=2 i=4: coin=1: dp[4]=min(inf, 1+dp[3])=3 coin=2: dp[4]=min(3, 1+dp[2])=2 dp[4]=2 i=5: coin=1: dp[5]=min(inf, 1+dp[4])=3 coin=2: dp[5]=min(3, 1+dp[3])=3 coin=5: dp[5]=min(3, 1+dp[0])=1 dp[5]=1 Result: dp[5]=1 ✓
💻 Solution
Python · Coin Change DP
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    
    for i in range(1, amount + 1):
        for coin in coins:
            if coin <= i:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    
    return dp[amount] if dp[amount] != float('inf') else -1
⌛ Complexity
Time: O(amount * len(coins)) Space: O(amount)

def word_break(s, wordDict):
    word_set = set(wordDict)
    dp = [False] * (len(s) + 1)

                dp[i] = True

    return dp[len(s)]

🔍 Step-by-Step Trace
n=1000, target=1 Number has 4 digits (n=1000: "1000") DFS(pos=0, sum=0, tight=True): pos=0 (leftmost): tight=True, max_digit=1 (from n's digit) Try digit 0: DFS(1, 0, False) — number < 1000 now Try digit 1: DFS(1, 1, True) — number = 1xxx DFS(1, 1, True): tight=True, max_digit=0 Try digit 0: DFS(2, 1, True) DFS(2, 1, True): tight=True, max_digit=0 Try digit 0: DFS(3, 1, True) DFS(3, 1, True): tight=True, max_digit=0 Try digit 0: DFS(4, 1, True) — all 4 digits done DFS(4, 1, True): pos=4 (end), sum=1, target=1? YES ✓ Count += 1 (number "1000") Also count from DFS(1, 0, False): Allows any digit, counts numbers with digit_sum=1: (all numbers <1000 with digit sum 1: 1, 10, 100) Result: 4 numbers total [1, 10, 100, 1000]
💻 Solution

def count_digit_sum(n, target):
    memo = {}
    
    def dfs(pos, digit_sum, tight, num_str):
        if pos == len(num_str):
            return 1 if digit_sum == target else 0
        
        if (pos, digit_sum, tight) in memo:
            return memo[(pos, digit_sum, tight)]
        
        max_digit = int(num_str[pos]) if tight else 9
        count = 0
        
        for digit in range(0, max_digit + 1):
            new_tight = tight and (digit == max_digit)
            count += dfs(pos + 1, digit_sum + digit, new_tight, num_str)
        
        memo[(pos, digit_sum, tight)] = count
        return count
    
    return dfs(0, 0, True, str(n))

Problem (LeetCode 1155): Count ways to roll d dice to get target sum. d=1, target=6 → 1 way (roll [6])
Digit DP Variant: State: (dice_count, current_sum). Transition: try each face 1-6. Memoize to avoid recomputing same state.
🔍 Step-by-Step Trace
d=2, target=7 Ways to roll 2 dice sum=7: [1,6], [2,5], [3,4], [4,3], [5,2], [6,1] Count = 6 DP approach: dp[0][0] = 1 (0 dice, sum 0) dp[i][s] = Σ dp[i-1][s-face] for face in 1..6 dp[1][1] = dp[0][0] = 1 dp[1][2] = dp[0][1] = 1 ... dp[1][6] = dp[0][5] = 1 (1 way to roll each face with 1 die) dp[2][7] = dp[1][6] + dp[1][5] + ... + dp[1][1] = 1 + 1 + 1 + 1 + 1 + 1 = 6 ✓
💻 Solution
Python · Dice Rolls DP
def num_rolls_to_target(d, target):
    MOD = 10**9 + 7
    memo = {}
    
    def dp(dice, s):
        if dice == 0:
            return 1 if s == 0 else 0
        if s <= 0:
            return 0
        
        if (dice, s) in memo:
            return memo[(dice, s)]
        
        count = 0
        for face in range(1, 7):
            count += dp(dice - 1, s - face)
        
        memo[(dice, s)] = count % MOD
        return memo[(dice, s)]
    
    return dp(d, target)

Traveling Salesman Problem (Bitmask DP)
Problem (LeetCode 943): Find shortest path visiting all cities exactly once (TSP). Return minimum distance.
Bitmask DP: State: dp[mask][i] = minimum cost to visit all cities in mask, ending at city i. Mask has n bits, each bit = city visited. Transition: try next unvisited city, update mask.
🔍 Step-by-Step Trace
4 cities, distance matrix: 0 1 2 3 0 0 10 15 20 1 10 0 35 25 2 15 35 0 30 3 20 25 30 0 mask=0001 (city 0): dp[0001][0] = 0 mask=0011 (cities 0,1): from city 0, go to 1: dp[0011][1] = dp[0001][0] + dist(0,1) = 10 mask=0111 (cities 0,1,2): from city 1, go to 2: dp[0111][2] = dp[0011][1] + dist(1,2) = 10 + 35 = 45 from city 0, go to 2: dp[0111][2] = min(45, dp[0101][0] + dist(0,2)) ...continue until mask=1111 (all cities) Final: min(dp[1111][i] + dist(i, 0)) for all ending cities i = shortest round trip
💻 Solution
Python · TSP Bitmask DP
def tsp(dist):
    n = len(dist)
    INF = float('inf')
    dp = [[INF] * n for _ in range(1 << n)]
    
    dp[1][0] = 0  # Start at city 0
    
    for mask in range(1 << n):
        for u in range(n):
            if dp[mask][u] == INF:
                continue
            # Try visiting next city
            for v in range(n):
                if mask & (1 << v) == 0:  # v not visited
                    new_mask = mask | (1 << v)
                    dp[new_mask][v] = min(dp[new_mask][v], dp[mask][u] + dist[u][v])
    
    # Return to city 0
    full_mask = (1 << n) - 1
    ans = min(dp[full_mask][i] + dist[i][0] for i in range(n))
    return ans
⌛ Complexity
Time: O(2^n * n²) Space: O(2^n * n)

Maximum Length of Pair Chain (Interval DP)
Problem (LeetCode 435): Find max pairs where each pair [a,b] can follow [c,d] if b < c. pairs=[[1,2],[7,8],[4,5]] → 3 (chain all three)
Greedy + Sorting OR DP: Sort by end position. Greedy: always pick pair with earliest end time (leaves room for next). DP: dp[i] = max chain length ending at pair i.
🔍 Step-by-Step Trace
pairs=[[1,2],[7,8],[4,5]] Sorted by end: [[1,2],[4,5],[7,8]] Greedy approach: Pick [1,2], end=2 Next: [4,5], start=4 > 2? YES, pick it, end=5 Next: [7,8], start=7 > 5? YES, pick it Chain: 3 pairs ✓ DP approach: dp[0] = 1 (pair [1,2]) dp[1] = max(1, 1 + 1) = 2 if [1,2]<[4,5] Yes, so dp[1] = 2 dp[2] = max(1, dp[1] + 1) = 3 if [4,5]<[7,8] Yes, so dp[2] = 3 Result: 3 ✓
💻 Solution
Python · Maximum Pair Chain
def max_chain_length(pairs):
    # Greedy: Sort by end position, pick earliest ending
    pairs.sort(key=lambda x: x[1])
    count = 0
    current_end = float('-inf')
    
    for start, end in pairs:
        if start > current_end:
            count += 1
            current_end = end
    
    return count

# Alternative DP approach:
def max_chain_length_dp(pairs):
    pairs.sort()
    n = len(pairs)
    dp = [1] * n
    
    for i in range(1, n):
        for j in range(i):
            if pairs[j][1] < pairs[i][0]:
                dp[i] = max(dp[i], dp[j] + 1)
    
    return max(dp)
⌛ Complexity
Time: O(n log n) greedy, O(n²) DP Space: O(1) greedy, O(n) DP
