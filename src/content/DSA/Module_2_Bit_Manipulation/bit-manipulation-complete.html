🧠 The Story — The Inventory System
Real World Story — Flags and Switches

You manage a system with 32 on/off features. You could use 32 separate boolean variables — or 1 integer.

Each bit in a 32-bit integer represents one feature flag. Bit 0 = feature_A, bit 1 = feature_B, etc.

Check if feature N is on: (flags >> N) & 1
Turn feature N on: flags | (1 << N)
Turn feature N off: flags & ~(1 << N)
Toggle feature N: flags ^ (1 << N)

This "bitmask as set" pattern powers algorithms: dynamic programming on subsets (bitmask DP), graph state compression, and many elegant interview solutions.

The Essential Bit Tricks
n & (n-1) → clear lowest set bit (used: is power of 2? count set bits)
n & (-n) → isolate lowest set bit (used: Fenwick Tree navigation)
n ^ n = 0 → XOR self = 0 (used: find single number)
a ^ b ^ a = b → XOR twice = identity (used: swap without temp)
~n + 1 = -n → two's complement negation
n >> 1 → divide by 2 (floor)
n << 1 → multiply by 2
Python · Core Bit Operations
# --- Check if power of 2 ---
def is_power_of_2(n): return n > 0 and (n & n-1) == 0
# Explanation: powers of 2 have exactly one set bit: 100...0
# n-1 flips all bits up to that bit: 011...1
# AND of them = 0 iff n is a power of 2

# --- Count set bits (Brian Kernighan's algorithm) ---
def count_bits(n):
    count = 0
    while n:
        n &= n - 1    # clear lowest set bit each iteration
        count += 1
    return count  # O(number of set bits) — faster than O(log n) for sparse

# --- Single Number: XOR all elements ---
from functools import reduce
from operator import xor
def single_number(nums): return reduce(xor, nums)
# All pairs cancel (a^a=0). Only unpaired number remains. O(n) time, O(1) space.

# --- Find two single numbers (appears exactly once each) ---
def two_singles(nums):
    xor_all = reduce(xor, nums)           # xor of the two unique numbers
    bit = xor_all & (-xor_all)            # isolate any differing bit
    a = b = 0
    for n in nums:
        if n & bit: a ^= n              # group by differing bit
        else:       b ^= n
    return [a, b]

# --- Reverse bits (32-bit integer) ---
def reverse_bits(n):
    result = 0
    for _ in range(32):
        result = (result << 1) | (n & 1)  # shift result left, add LSB of n
        n >>= 1                           # remove LSB from n
    return result

Python · Counting Bits DP and Subset Sum
# --- Counting Bits: for i in [0,n], count set bits --- O(n)
def count_bits_all(n):
    dp = [0] * (n + 1)
    for i in range(1, n+1):
        dp[i] = dp[i >> 1] + (i & 1)  # dp[i] = dp[i//2] + (i is odd)
    return dp

# --- Sum of All XOR Subsets: enumerate all 2^n subsets --- O(2^n)
def subset_xor_sum(nums):
    n, total = len(nums), 0
    for mask in range(1 << n):   # iterate all 2^n subsets
        xor_val = 0
        for i in range(n):
            if mask & (1 << i):    # bit i is set: include nums[i]
                xor_val ^= nums[i]
        total += xor_val
    return total

Missing Number
Problem (LeetCode 268): Array contains n distinct numbers taken from 0 to n. Find the missing number. E.g. [3,0,1] → 2.
XOR trick: XOR all numbers from 0 to n, then XOR all array elements. Every number that appears in BOTH cancels out (a^a=0). Only the missing number remains, since it's only in the range [0,n] but not in the array.
🔍 Step-by-Step Trace
nums = [3, 0, 1] n=3 Method 1 — XOR: XOR(0..n) = 0^1^2^3 = 0 XOR(nums) = 3^0^1 = 2 Combined: 0^1^2^3 ^ 3^0^1 = (0^0)^(1^1)^(3^3)^2 = 2 ✓ (each paired number cancels, only 2 is unpaired) Method 2 — Math: expected_sum = n*(n+1)/2 = 3*4/2 = 6 actual_sum = 3+0+1 = 4 missing = 6-4 = 2 ✓ XOR is better: no overflow risk, equally O(n) time, O(1) space
💻 Solution
Python · Missing Number
def missing_number(nums):
    result = len(nums)                # start with n
    for i, n in enumerate(nums):
        result ^= i ^ n              # XOR with both index and value
    return result
    # result = (0^0)^(1^1)^...^(missing^n)^... = missing
⌛ Complexity
Time: O(n) Space: O(1)

Number of 1 Bits (Hamming Weight)
Problem (LeetCode 191): Count the number of set bits (1s) in the binary representation of an integer. E.g. n=00000000000000000000000000001011 → 3.
Brian Kernighan's algorithm: n & (n-1) clears the LOWEST set bit of n. Count how many times you can do this before n reaches 0. This is O(k) where k=number of set bits, faster than O(32) for sparse integers.
🔍 Step-by-Step Trace
n = 1011 (binary) = 11 (decimal) n=1011: 1011 & 1010 = 1010 count=1 (cleared lowest '1' at position 0) n=1010: 1010 & 1001 = 1000 count=2 (cleared '1' at position 1) n=1000: 1000 & 0111 = 0000 count=3 (cleared '1' at position 3) n=0000: stop Answer = 3 ✓ Why n & (n-1) clears lowest bit: n = ...1000 (some bits)...1(target)000...0 n-1 = ...1000 (same bits)...0 111...1 AND = ...1000 (same bits)...0 000...0 → lowest '1' and all bits below it cleared
💻 Solution
Python · Number of 1 Bits
def hamming_weight(n):
    count = 0
    while n:
        n &= n - 1     # clear lowest set bit
        count += 1
    return count

# Counting Bits variant (LeetCode 338): all counts 0 to n in O(n)
def count_bits(n):
    dp = [0] * (n+1)
    for i in range(1, n+1):
        dp[i] = dp[i >> 1] + (i & 1)  # right shift + check last bit
    return dp
⌛ Complexity
Time: O(k) — k set bits Space: O(1)
3
Add Two Integers Without + or -
Problem (LeetCode 371): Add two integers a and b without using + or - operators. Must handle negative numbers.
Bit addition simulation: a^b gives the bits where they differ (sum without carry). a&b gives carry positions. Left-shift carry by 1 (carry propagates left). Repeat until no carry. Python requires 32-bit mask for negative numbers.
🔍 Step-by-Step Trace
a=2 (0010) b=3 (0011) Round 1: sum_bits = a^b = 0010^0011 = 0001 carry = a&b = 0010&0011 = 0010, shift left = 0100 (=4) a=0001, b=0100 Round 2: sum_bits = 0001^0100 = 0101 (=5) carry = 0001&0100 = 0000, shift left = 0000 a=0101, b=0000 b=0 → no more carry → return a=0101=5 ✓ Example with carry chain: a=7(0111) b=1(0001) sum=0110, carry=0001<<1=0010 sum=0100, carry=0010<<1=0100 sum=0000, carry=0100<<1=1000 sum=1000=8 ✓
💻 Solution
Python · Add Without Plus
def get_sum(a, b):
    MASK = 0xFFFFFFFF
    MAX  = 0x7FFFFFFF
    while b != 0:
        carry = (a & b) & MASK             # carry positions (masked)
        a     = (a ^ b) & MASK             # sum without carry (masked)
        b     = (carry << 1) & MASK        # shift carry left
    # Handle negative numbers (Python's arbitrary int)
    return a if a <= MAX else ~(a ^ MASK)
⌛ Complexity
Time: O(log max_val) — carry propagation Space: O(1)
