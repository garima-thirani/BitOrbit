🧠 The Story — The Train
Real World Story — Coupled Train Cars

A linked list is a train. Each car holds cargo (data) and a coupling to the next car.

Adding a car in the middle: uncouple two adjacent cars, insert new car, reconnect — O(1) given the right pointer.
Removing a car: point the previous car's coupling to the car AFTER the one being removed.

The key difference from arrays: No random access (you can't jump to car 47 directly), but insertions and deletions anywhere are O(1) given a pointer to the right place.

The dummy node trick: Add a fake head node at the start. This eliminates special cases for head deletion/insertion. Every operation becomes uniform. Use this in ALL linked list problems.

The Dummy Node Trick — Eliminates Edge Cases

Instead of handling "what if head is removed?" as a special case, create a dummy node whose next points to head. After all operations, return dummy.next.

A linked list is memory stitched together by pointers. There is no random access — you follow threads. This single constraint shapes every algorithm: you must keep a pointer to where you were before you move, because you can never go back. The three-pointer reversal (prev / curr / next) is the atom of most list problems; Floyd’s tortoise-and-hare turns the no-random-access constraint into a feature — two pointers at different speeds will meet in a cycle. Dummy head eliminates the special case when head itself gets deleted or replaced.

🩸 Three-pointer reversal

Save nxt = curr.next BEFORE you overwrite it. Then curr.next = prev to flip the arrow, and slide both pointers forward. Repeat until curr is None; prev is the new head.

🐢 Floyd’s cycle detection

Slow moves 1 step, fast moves 2. If there's a cycle they must meet; if there's no cycle fast hits None first. Meeting point math also gives the cycle entry.

👤 Dummy head

Prepend a sentinel node before the real head. Deletes, merges, and partitions work without “is head the special case?” branches. Return dummy.next at the end.

⏱️ Fast & slow for middle

When fast hits the tail, slow is at the middle. Used as a sub-step for palindrome check, merge sort on lists, and reorder list.

reverse: prev=None; while curr: nxt=curr.next; curr.next=prev; prev=curr; curr=nxt
cycle: slow+=1, fast+=2; if slow==fast → cycle. To find entry: reset one to head, both move 1 step, they meet at entry
middle: while fast.next and fast.next.next: slow+=1, fast+=2

🌍 Where this lives in the real world
  • Browser history (back/forward): a doubly-linked list — prev/next pointers give O(1) navigation.
  • Undo/redo in editors: each action is a node; undo traverses prev, redo follows next.
  • OS process scheduler: a circular linked list (round-robin) — cycle detection keeps the scheduler from running forever on a zombie process.
  • LRU cache (doubly-linked + hashmap): the most-recently-used node is always the head; eviction pulls from the tail.
  • Music playlist (shuffle/loop): next/previous songs are pointer traversals; a looped playlist is a circular linked list.
  • Memory allocator free-list: freed blocks are linked; the allocator walks the list to find a big-enough gap.
⚠️ The traps

Save next before overwriting: the first line of every reversal loop is nxt = curr.next. Forget it and you've lost the rest of the list forever. Null checks before .next.next: fast.next.next crashes if fast.next is None — always guard fast and fast.next. Off-by-one on middle: while fast.next and fast.next.next leaves slow on the first middle node; change to while fast and fast.next for the second middle — know which variant you need before you write it.

🎬 Reversal — The Most Common Interview Operation

Initial: None 1 → 2 → 3 → 4 → None prev=None, curr=1 Step 1: next=2, 1→None, prev=1, curr=2 Step 2: next=3, 2→1, prev=2, curr=3 Step 3: next=4, 3→2, prev=3, curr=4 Step 4: next=None, 4→3, prev=4, curr=None Return prev=4: 4→3→2→1→None ✓
Python · Reversal, Cycle Detection, Merge
# --- Reverse Linked List: O(n) time, O(1) space ---
def reverse(head):
    prev = None
    while head:
        nxt  = head.next    # save next before overwriting
        head.next = prev    # reverse the link
        prev = head         # advance prev
        head = nxt          # advance head
    return prev             # prev is the new head

# --- Detect Cycle: Floyd's Two Pointers ---
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast: return True
    return False

# --- Merge Two Sorted Lists ---
def merge_sorted(l1, l2):
    dummy = ListNode(0)
    cur = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            cur.next = l1; l1 = l1.next    # take from l1
        else:
            cur.next = l2; l2 = l2.next    # take from l2
        cur = cur.next
    cur.next = l1 or l2                  # attach remaining
    return dummy.next

# --- Find Middle (used for merge sort, palindrome check) ---
def find_middle(head):
    slow = fast = head
    while fast.next and fast.next.next:
        slow = slow.next        # slow moves 1 step
        fast = fast.next.next   # fast moves 2 steps
    return slow                 # slow = middle
🎬 3-pointer reversal — flip each arrow, one node at a time
The three pointers work as a relay baton. nxt saves the forward path before you cut it; curr.next = prev flips the arrow; then both prev and curr slide one step right. After the loop, curr is None and prev is pointing at the old tail — now the new head. ▶ loops every 11s.

🎬 Palindrome Check — Classic Multi-Step Problem
📖
Algorithm: (1) Find middle. (2) Reverse second half. (3) Compare first and second half node by node. (4) Optionally restore original list.
List: 1→2→3→2→1 Step 1: middle = node(3) Step 2: reverse second half: 1→2←3 & 2→1→None becomes 1→2→None First half: 1→2→3 Second half (reversed): 1→2→None Step 3: compare: 1==1, 2==2 → PALINDROME ✓
Python · Reverse K-Group & LRU Cache
# --- Reverse Nodes in K-Group (Hard) ---
def reverse_k_group(head, k):
    # Check if k nodes remain
    check, count = head, 0
    while check and count < k: check = check.next; count += 1
    if count < k: return head   # not enough nodes: don't reverse

    prev, curr = None, head
    for _ in range(k):
        nxt = curr.next; curr.next = prev; prev = curr; curr = nxt

    head.next = reverse_k_group(curr, k)   # head is now tail of reversed group
    return prev

# --- LRU Cache: OrderedDict (Python built-in) ---
from collections import OrderedDict
class LRUCache:
    def __init__(self, capacity): self.cap = capacity; self.cache = OrderedDict()
    def get(self, key):
        if key not in self.cache: return -1
        self.cache.move_to_end(key)       # mark as recently used
        return self.cache[key]
    def put(self, key, value):
        if key in self.cache: self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.cap: self.cache.popitem(last=False)  # evict LRU
🎬 Floyd’s tortoise & hare — two speeds, one meeting point
Two runners, one ring. If there’s a cycle, the fast runner laps the slow one inside it — their distance mod cycle-length decreases by 1 each step until it hits 0. If there’s no cycle, fast exits the list. No visited set needed, O(1) space. ▶ loops every 12s.

1
Remove Nth Node From End
Problem (LeetCode 19): Given the head of a linked list, remove the Nth node from the end in ONE pass. E.g. 1→2→3→4→5 with n=2 → 1→2→3→5.
Two-pointer gap trick: Use a dummy head to handle edge cases. Move fast pointer N steps ahead, then move both until fast.next is None. Now slow.next is the node to delete. The N-step gap means when fast reaches the last node, slow is exactly at the node before the target.
🔍 Step-by-Step Trace
List: dummy→1→2→3→4→5 n=2 Step 1: Advance fast by N=2 steps: fast starts at dummy fast after 1 step: →1 fast after 2 steps: →2 fast = node(2), slow = dummy Step 2: Move both until fast.next is None: fast=2, slow=dummy → fast=3, slow=1 fast=3, slow=1 → fast=4, slow=2 fast=4, slow=2 → fast=5, slow=3 fast.next=None, STOP slow = node(3), slow.next = node(4) → DELETE node(4) slow.next = node(4).next = node(5) Result: dummy→1→2→3→5 ✓
💻 Solution
Python · Remove Nth From End
def remove_nth_from_end(head, n):
    dummy = ListNode(0)
    dummy.next = head
    fast = slow = dummy

    for _ in range(n):         # advance fast by n steps
        fast = fast.next

    while fast.next:           # move both until fast at last node
        fast = fast.next
        slow = slow.next

    slow.next = slow.next.next  # delete target node
    return dummy.next
⌛ Complexity
Time: O(n) Space: O(1)
2
Intersection of Two Linked Lists
Problem (LeetCode 160): Find the node where two singly linked lists intersect. If they don't intersect, return null. Must use O(1) memory.
Path equalization trick: Pointers a and b start at heads. When one reaches end, redirect to the OTHER list's head. After at most len(A)+len(B) steps, they'll meet at the intersection (or both be None). The key: both pointers travel the same total distance.
🔍 Step-by-Step Trace
List A: a1→a2→c1→c2→c3 List B: b1→b2→b3→c1→c2→c3 (shared: c1→c2→c3) Pointer a: a1→a2→c1→c2→c3→None→b1→b2→b3→[c1] Pointer b: b1→b2→b3→c1→c2→c3→None→a1→a2→[c1] Both reach c1 simultaneously! Distance for a: 2+3 + 3 = 8 Distance for b: 3+3 + 2 = 8 ← same! No intersection case: Both walk len(A)+len(B) steps and both end up at None → return None
💻 Solution
Python · Linked List Intersection
def get_intersection_node(headA, headB):
    a, b = headA, headB
    while a != b:               # pointers are equal at intersection or None
        a = a.next if a else headB   # redirect to other list at end
        b = b.next if b else headA
    return a                     # intersection node or None
⌛ Complexity
Time: O(m+n) Space: O(1)
3
Sort List — Merge Sort
Problem (LeetCode 148): Sort a linked list in O(n log n) time and O(1) extra space. The challenge: arrays can use Timsort in-place, but linked lists require pointer manipulation.
Bottom-up merge sort: Find the middle using fast/slow pointers, split into two halves, recursively sort each, then merge. Merge sort is ideal for linked lists because it accesses nodes sequentially (no random access needed). Merge two sorted lists is O(n).
🔍 Step-by-Step Trace
List: 4→2→1→3 Split: find mid slow=4, fast=4 slow=2, fast=1 (fast.next.next=None → stop) mid = 2, split: [4→2] and [1→3] Recursively sort [4→2]: split: [4] and [2] merge [4],[2] → [2→4] Recursively sort [1→3]: split: [1] and [3] merge [1],[3] → [1→3] Merge [2→4] and [1→3]: compare 2 vs 1: take 1, result=[1], l2=[3] compare 2 vs 3: take 2, result=[1→2], l1=[4] compare 4 vs 3: take 3, result=[1→2→3], l2=None append 4: result=[1→2→3→4] ✓
💻 Solution
Python · Sort List (Merge Sort)
def sort_list(head):
    if not head or not head.next: return head

    # Find middle and split
    slow, fast = head, head.next
    while fast and fast.next:
        slow = slow.next; fast = fast.next.next
    mid = slow.next
    slow.next = None             # cut the list

    left  = sort_list(head)
    right = sort_list(mid)

    # Merge two sorted halves
    dummy = cur = ListNode(0)
    while left and right:
        if left.val <= right.val: cur.next = left;  left  = left.next
        else:                     cur.next = right; right = right.next
        cur = cur.next
    cur.next = left or right   # attach remaining nodes
    return dummy.next
⌛ Complexity
Time: O(n log n) Space: O(log n) stack
