🧠 The Story — The Scoreboard
Real World Story — Game Scores

You have a scoreboard with daily scores. "What's the total score from day 3 to day 7?" — 100 times a day.

Naive: Scan days 3 to 7 each time. O(n) per query. 100 queries = 100n operations.

Prefix Sum: Precompute prefix[i] = sum of scores[0..i]. Then query(3,7) = prefix[7] - prefix[2]. O(1) per query after O(n) preprocessing.

This "precompute once, query many times" pattern is the essence of the prefix sum technique — and it appears in dozens of interview problems.

The Core Array Patterns
  • Prefix Sum: Precompute cumulative sums for range queries
  • Two Pointers: Reduce O(n^2) to O(n) for pair/triplet problems
  • Sliding Window: Find optimal contiguous subarray
  • Kadane's Algorithm: Maximum subarray in O(n)
  • Boyer-Moore Voting: Find majority element in O(1) space
  • Binary Search on Answer: "Find minimum k such that condition holds"
💡 The Deep Intuition — Trade a Second Loop for a Single Memory

Almost every "fast array" trick is the same move in disguise: the brute-force solution re-walks the array again and again; the fast one walks it once and remembers the right summary as it goes. Prefix sums remember a running total. Two pointers remember where the other end is. Sliding windows remember the current window's contents. Kadane remembers the best subarray ending here. Once you see "I'm recomputing something I already knew," you've found the optimization.

🟡 Prefix sum — precompute once

When you'll answer many range-sum questions, build a cumulative array first. Then any range is one subtraction: a re-scan per query (O(n)) collapses to O(1). Generalises to prefix-XOR, prefix-count, and 2D prefix sums for submatrix queries.

🔵 Two pointers — squeeze from both ends

On sorted data, a left and right pointer let you decide which way to move from the current sum: too small → move left up, too big → move right down. Each pointer only ever advances, so the whole scan is O(n) instead of O(n²).

🟣 Kadane — the local/global trick

The maximum subarray ends somewhere. So ask at each index: "best sum ending here?" Either extend the previous best or restart fresh:

curr = max(num, curr + num)
best = max(best, curr)

🟢 Boyer-Moore — cancel the noise

A true majority (> n/2) survives if every other value cancels one copy of it. Keep a candidate and a count; pair off opposites. Whatever is left standing must be the majority — O(n) time, O(1) space, almost magical.

🌍 Where these array tricks earn their keep
  • Spreadsheets & dashboards: a "running total" column is a prefix sum; "revenue from Q2 to Q3" is one subtraction, not a re-add.
  • Your bank statement: the balance after each transaction is a prefix sum of deposits and withdrawals.
  • Image processing (integral images): 2D prefix sums let face-detection sum any rectangle of pixels in O(1) — the trick behind real-time camera filters.
  • Stock analysis: "best buy-low/sell-high window" is exactly Kadane's maximum-subarray on daily price changes.
  • Voting & streaming telemetry: Boyer-Moore finds the dominant signal in one pass over a firehose, using almost no memory.
⚠️ Classic off-by-one & ordering traps

Prefix bounds: with a size-(n+1) prefix array, range [l, r] inclusive is prefix[r+1] - prefix[l] — mixing these up is the #1 bug. Two pointers need sorted input — forgetting to sort first silently returns wrong pairs. Kadane with all-negative arrays: initialise best to the first element (or −∞), not 0, or you'll wrongly return 0 for [-3,-1,-2].

Python · Prefix Sum, Kadane, Boyer-Moore
# --- Prefix Sum: range sum in O(1) ---
def build_prefix(nums):
    prefix = [0] * (len(nums) + 1)
    for i, v in enumerate(nums):
        prefix[i+1] = prefix[i] + v
    return prefix
# query(l, r) = prefix[r+1] - prefix[l]

# --- Kadane's: Maximum Subarray O(n) ---
def max_subarray(nums):
    max_sum = curr = nums[0]
    for num in nums[1:]:
        curr = max(num, curr + num)   # extend or restart subarray
        max_sum = max(max_sum, curr)
    return max_sum

# --- Boyer-Moore Voting: Majority Element O(n) time, O(1) space ---
def majority_element(nums):
    count, candidate = 0, None
    for num in nums:
        if count == 0: candidate = num
        count += 1 if num == candidate else -1
    return candidate   # guaranteed >n/2, so always survives cancellation

# --- Product of Array Except Self: O(n), O(1) space ---
def product_except_self(nums):
    n = len(nums)
    result = [1] * n
    for i in range(1, n):
        result[i] = result[i-1] * nums[i-1]   # left products
    right = 1
    for i in range(n-1, -1, -1):
        result[i] *= right                     # multiply right products
        right *= nums[i]
    return result
🎬 Prefix sum — build cumulative totals once, answer any range in O(1)
Pay O(n) once, then every query is O(1). prefix[i+1] = prefix[i] + nums[i] stores the running total; the sum of any window is the difference of its two endpoints. A thousand range queries that would cost a thousand re-scans become a thousand subtractions. ▶ loops every 11s.
🎬 Kadane's algorithm — extend the run, or cut your losses and restart
The whole algorithm is one decision, repeated. At each element: is the running sum still helping? If curr + num < num, the past is dead weight — throw it away and start a fresh run here. A single O(n) pass, O(1) memory. ▶ loops every 12s.

Python · 3Sum, Container With Most Water, Jump Game
# --- 3Sum ---
def three_sum(nums):
    nums.sort(); result = []
    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i-1]: continue  # skip dups
        l, r = i+1, len(nums)-1
        while l < r:
            s = nums[i] + nums[l] + nums[r]
            if   s == 0:
                result.append([nums[i], nums[l], nums[r]])
                while l < r and nums[l] == nums[l+1]: l += 1  # skip dups
                while l < r and nums[r] == nums[r-1]: r -= 1
                l += 1; r -= 1
            elif s < 0: l += 1
            else:       r -= 1
    return result

# --- Jump Game I: can you reach end? ---
def can_jump(nums):
    reach = 0
    for i, jump in enumerate(nums):
        if i > reach: return False        # can't reach this position
        reach = max(reach, i + jump)    # update farthest reachable
    return True
🎬 3Sum — fix one number, then squeeze two pointers toward each other
Sorting is what unlocks the two pointers. Because the array is ordered, the sum tells you which pointer to move: too small, raise the floor; too big, lower the ceiling. The inner scan is O(n), so the whole 3Sum is O(n²) instead of the brute-force O(n³). ▶ loops every 11s.

Next Permutation
Problem (LeetCode 31): Find the next lexicographically greater permutation of array. If already the largest, rearrange to smallest (ascending order). Must modify in-place.
Three-step algorithm: (1) Find rightmost index i where nums[i] < nums[i+1] — this is the 'descent' point. (2) Find smallest element to the right of i that is greater (rightmost j where nums[j]>nums[i]). (3) Swap i and j, then reverse everything after i.
🔍 Step-by-Step Trace
nums = [1,2,3,2,1] Step 1 — Find rightmost ascent: scan from right: nums[3]=2 < nums[2]=3? No. nums[2]=3 > nums[1]=2? Yes. Wait: find where nums[i] < nums[i+1] FROM RIGHT: i=3: nums[3]=2 > nums[4]=1 → no i=2: nums[2]=3 > nums[3]=2 → no i=1: nums[1]=2 < nums[2]=3 → YES! i=1 Step 2 — Find smallest element > nums[1]=2 to the right: nums[2]=3, nums[3]=2, nums[4]=1 Rightmost > 2: j=2 (value 3) Step 3 — Swap i=1 and j=2: [1,3,2,2,1] Reverse suffix after i=1: reverse [2,2,1] → [1,2,2] Result: [1,3,1,2,2] ✓
💻 Solution
Python · Next Permutation
def next_permutation(nums):
    n = len(nums)
    i = n - 2
    # Step 1: find rightmost ascent
    while i >= 0 and nums[i] >= nums[i+1]: i -= 1

    if i >= 0:
        # Step 2: find rightmost element > nums[i]
        j = n - 1
        while nums[j] <= nums[i]: j -= 1
        nums[i], nums[j] = nums[j], nums[i]   # Step 2: swap

    # Step 3: reverse suffix (make it ascending)
    nums[i+1:] = nums[i+1:][::-1]
⌛ Complexity
Time: O(n) Space: O(1)

Find Duplicate — Floyd's Cycle Detection
Problem (LeetCode 287): Array of n+1 integers where each is in [1,n]. Exactly one duplicate exists. Find it without modifying array and using O(1) extra space.
Floyd's cycle detection disguised as an array problem: View the array as a linked list where nums[i] points to the next node. Since values are in [1,n] and there are n+1 elements, a duplicate value means two nodes point to the same next node — creating a cycle. Find the cycle start = the duplicate.
🔍 Step-by-Step Trace
nums = [1,3,4,2,2] (index 0→1, 1→3, 2→4, 3→2, 4→2) Visualize as linked list: 0→1→3→2→4→2→4→... (cycle!) Phase 1 — find intersection (slow and fast pointers): slow=nums[0]=1, fast=nums[nums[0]]=nums[1]=3 slow=nums[1]=3, fast=nums[nums[3]]=nums[2]=4 slow=nums[3]=2, fast=nums[nums[4]]=nums[2]=4 slow=nums[2]=4, fast=nums[nums[4]]=nums[2]=4 slow==fast=4 → intersection found! Phase 2 — find cycle start: Reset one pointer to 0 (start): ptr1=0, ptr2=4 ptr1=nums[0]=1, ptr2=nums[4]=2 ptr1=nums[1]=3, ptr2=nums[2]=4 ptr1=nums[3]=2, ptr2=nums[4]=2 ptr1==ptr2=2 → cycle start = DUPLICATE = 2 ✓
💻 Solution
Python · Find Duplicate (Floyd's)
def find_duplicate(nums):
    # Phase 1: detect cycle (find intersection)
    slow = fast = nums[0]
    while True:
        slow = nums[slow]
        fast = nums[nums[fast]]
        if slow == fast: break

    # Phase 2: find cycle start (= duplicate)
    ptr1, ptr2 = 0, slow
    while ptr1 != ptr2:
        ptr1 = nums[ptr1]
        ptr2 = nums[ptr2]
    return ptr1
⌛ Complexity
Time: O(n) Space: O(1)

Maximum Product Subarray
Problem (LeetCode 152): Find the contiguous subarray with the largest product. E.g. [2,3,-2,4] → 6 ([2,3]).
Track both min and max simultaneously: Unlike max sum (Kadane's), products can flip sign. A very negative product times a negative number becomes a large positive. So at each step, track BOTH current max and current min. Swap them when hitting a negative number.
🔍 Step-by-Step Trace
nums = [2, 3, -2, 4] curr_max=1 curr_min=1 global_max=negative_inf i=0 n=2: new_max = max(2, 1*2, 1*2) = 2 new_min = min(2, 1*2, 1*2) = 2 curr_max=2, curr_min=2, global_max=2 i=1 n=3: new_max = max(3, 2*3, 2*3) = 6 new_min = min(3, 2*3, 2*3) = 3 curr_max=6, curr_min=3, global_max=6 i=2 n=-2: new_max = max(-2, 6*(-2), 3*(-2)) = max(-2,-12,-6) = -2 new_min = min(-2, 6*(-2), 3*(-2)) = min(-2,-12,-6) = -12 curr_max=-2, curr_min=-12, global_max=6 i=3 n=4: new_max = max(4, -2*4, -12*4) = max(4,-8,-48) = 4 new_min = min(4, -2*4, -12*4) = -48 global_max = max(6, 4) = 6 ✓
💻 Solution
Python · Maximum Product Subarray
def max_product(nums):
    curr_max = curr_min = 1
    global_max = float('-inf')

    for n in nums:
        # All three candidates for the new max
        candidates = (n, curr_max*n, curr_min*n)
        curr_max = max(candidates)
        curr_min = min(candidates)
        global_max = max(global_max, curr_max)
    return global_max
⌛ Complexity
Time: O(n) Space: O(1)
