🧠 The Story — The Kitchen
Real World Story — Plate Stack & Order Queue

A restaurant: plates are stacked (last in, first out — LIFO). Orders are queued (first in, first out — FIFO).

Stack (LIFO): Push a plate on top. Take from top. The plate you just placed is the one you grab. Used for: undo/redo, browser back button, function call stack, expression evaluation, DFS.

Queue (FIFO): Customer orders arrive in sequence. First order served first. Used for: BFS, print spooler, task scheduler, streaming data.

Deque (Double-ended): Plates you can add/remove from BOTH ends. Used for: sliding window maximum, palindrome check, monotonic deque problems.

Stack and queue are the same ingredient cooked two different ways. Both store a sequence; the only difference is which end you eat from. LIFO (stack) says: the last thing in is the first thing out — the call stack, undo history, browser back button, and DFS all live here. FIFO (queue) says: the first thing in is the first thing out — BFS, task schedulers, and print spoolers live here. The monotonic stack is the most powerful interview pattern: it answers “next/previous greater or smaller element” for every position in a single O(n) pass, by guaranteeing the stack is always sorted.

📖 Stack (LIFO) use-cases

  • Valid parentheses, expression evaluation
  • DFS, recursion simulation
  • Undo/redo, browser history
  • Monotonic stack: next greater element, histogram, buildings with views

📤 Queue (FIFO) use-cases

  • BFS level-order traversal
  • Task scheduler, rate limiter
  • Sliding window max (monotonic deque)
  • Two-stack queue: amortized O(1) enqueue & dequeue

📉 Monotonic stack template

For each element: pop all stack elements that violate the monotone property. The popped element found its answer (“next greater” = current element). Then push current index.

⚡ Python choices

Stack: list (append/pop). Queue: collections.deque (append/popleft). Never list.pop(0) for a queue — O(n) shift.

next greater element: while stack and a[stack[-1]] < current: pop and record → push index
histogram max rect: monotonic increasing stack of indices; pop when shorter bar found, width = gap to new top
sliding window max: monotonic decreasing deque of indices; remove expired (front) and smaller (back) on each step

🌍 Where this lives in the real world
  • CPU call stack: every function call pushes a frame; return pops it. Stack overflow is a literal stack overflow.
  • Undo in editors: Ctrl+Z pops the last action; Ctrl+Y pushes it to a redo stack.
  • Web server request queue: incoming requests are enqueued; workers dequeue FIFO to avoid starvation.
  • Stock span / price analysis: “how many consecutive days was the price lower than today?” — monotonic stack in financial data.
  • Browser back button: each page visit pushes to a stack; “back” pops it.
  • Compiler syntax checking: balanced parentheses / brackets / braces is the classic stack validity check.
⚠️ The traps

list.pop(0) for a queue: shifting all elements left is O(n) per dequeue — use deque.popleft() which is O(1). Forgetting indices in monotonic stack: you need to store indices, not values, because the width calculation (i - stack[-1] - 1) requires knowing where the previous shorter bar was. Histogram sentinel: append a 0 bar at the end to flush all remaining stack entries — otherwise elements that never found a shorter right neighbour are silently ignored.

Python · Stack, Queue, Deque — API
# --- Stack: use list ---
stack = []
stack.append(1)         # push
stack.append(2)
top = stack[-1]         # peek: O(1)
val = stack.pop()       # pop: O(1)

# --- Queue: use collections.deque ---
from collections import deque
queue = deque()
queue.append(1)         # enqueue: O(1)
queue.append(2)
front = queue[0]        # peek front: O(1)
val = queue.popleft()   # dequeue: O(1)  ← use deque, NOT list.pop(0) which is O(n)!

# --- Deque ---
dq = deque()
dq.appendleft(1)        # add to front
dq.append(2)            # add to back
dq.popleft()            # remove from front
dq.pop()                # remove from back
🎬 LIFO vs FIFO — the same push, a different exit door
Same data, opposite order. A stack exits from the top (where you entered), so the newest item comes out first. A queue exits from the other end, so the oldest item comes out first. Which door you exit through changes everything about the traversal order. ▶ loops every 11s.

🎬 Monotonic Stack — The Power Pattern
The Core Insight

A monotonic stack is a stack that maintains elements in monotonically increasing or decreasing order.

When to use: "Find the nearest greater/smaller element to the left/right of each element." Any problem with "next greater", "previous smaller", "span", "buildings with views", "histogram".

Template: For each element, pop all elements from stack that are smaller (or larger). The element you're about to push found its "previous greater" in the current stack top. The popped elements found their "next greater" in the current element.

Python · Monotonic Stack — Daily Temperatures
# Problem: for each day, how many days until a warmer temperature?
# temperatures=[73,74,75,71,69,72,76,73] → [1,1,4,2,1,1,0,0]
def daily_temperatures(T):
    result = [0] * len(T)
    stack = []  # stack of indices (decreasing temperatures)

    for i, temp in enumerate(T):
        while stack and T[stack[-1]] < temp:   # found warmer day for stack top
            j = stack.pop()
            result[j] = i - j                    # days to wait
        stack.append(i)                          # push current day's index

    return result

# i=0: stack=[0]
# i=1: T[0]=73 < 74 → result[0]=1, stack=[1]

# i=3: 71 < 75? No → stack=[2,3]
# i=4: 69 < 71? No → stack=[2,3,4]

🎬 Monotonic stack — each popped index found its "next warmer day"
The stack keeps "unanswered questions". Each index on the stack is waiting for a warmer day. When a warmer day arrives it answers all the questions it can and then queues itself as a new unanswered question. Every index enters and exits the stack at most once — O(n) total. ▶ loops every 13s.

🎬 Largest Rectangle in Histogram
📏
Story: For each bar, find the maximum width rectangle that can be formed using this bar as the SHORTEST bar. That width = (first shorter bar to right) - (first shorter bar to left) - 1. Monotonic stack gives both boundaries in O(n).
heights=[2,1,5,6,2,3] When we pop 6 (because 2 is shorter): right_bound=4(index of 2), left_bound=2(stack top=5's index) Area = 6*(4-2-1)=6*1=6 When we pop 5: right_bound=4, left_bound=1(index of 1) Area = 5*(4-1-1)=5*2=10 ← MAXIMUM Final answer = 10 ✓
Python · Largest Rectangle & Valid Parentheses
# --- Largest Rectangle in Histogram ---
def largest_rectangle(heights):
    stack, max_area = [], 0
    heights.append(0)                        # sentinel 0 to flush stack at end
    for i, h in enumerate(heights):
        while stack and heights[stack[-1]] > h:
            height = heights[stack.pop()]
            width  = i if not stack else i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)
    return max_area

# --- Valid Parentheses ---
def is_valid(s):
    stack = []
    pairs = {')':'(', ']':'[', '}':'{'}
    for ch in s:
        if ch in '([{': stack.append(ch)
        elif not stack or stack[-1] != pairs[ch]: return False
        else: stack.pop()
    return not stack

# --- Implement Queue Using Two Stacks ---
class MyQueue:
    def __init__(self): self.inbox = []; self.outbox = []
    def push(self, x): self.inbox.append(x)
    def pop(self):
        self.peek()
        return self.outbox.pop()
    def peek(self):
        if not self.outbox:              # transfer all at once: amortized O(1)
            while self.inbox: self.outbox.append(self.inbox.pop())
        return self.outbox[-1]
    def empty(self): return not self.inbox and not self.outbox
🎬 Largest rectangle in histogram — monotonic stack pins each bar’s left boundary
Every bar is a rectangle waiting to know its width. A monotone-increasing stack keeps track of bars whose left boundary is confirmed. When a shorter bar arrives it triggers all taller pending bars to compute their area (right boundary = current index, left boundary = new stack top). The sentinel 0 at the end flushes any remaining bars. ▶ loops every 12s.

Min Stack — O(1) getMin
Problem (LeetCode 155): Design a stack that supports push, pop, top, and getMin operations all in O(1) time.
Auxiliary min stack: Keep a secondary stack that tracks the running minimum. When pushing a value ≤ current min, push it to min_stack too. When popping, if the popped value equals min_stack's top, pop min_stack too. The min_stack always has the minimum for the current state of the main stack.
🔍 Step-by-Step Trace
Operations: push(5), push(3), push(7), push(2), pop(), getMin() push(5): stack=[5], min_stack=[5] (5 <= empty, push to min) push(3): stack=[5,3], min_stack=[5,3] (3 <= 5, push to min) push(7): stack=[5,3,7], min_stack=[5,3] (7 > 3, DON'T push to min) push(2): stack=[5,3,7,2], min_stack=[5,3,2] (2 <= 3, push to min) pop(): remove 2, 2==min_stack.top(2) → pop min_stack too stack=[5,3,7], min_stack=[5,3] getMin(): min_stack.top() = 3 ✓ Alternative: store (val, min_at_this_point) tuples: push(5) → (5,5), push(3) → (3,3), push(7) → (7,3), push(2) → (2,2) getMin() = current tuple's second value = 2
💻 Solution
Python · Min Stack
class MinStack:
    def __init__(self):
        self.stack = []
        self.min_stack = []

    def push(self, val):
        self.stack.append(val)
        # Track min: push if min_stack empty OR val is new min
        cur_min = self.min_stack[-1] if self.min_stack else float('inf')
        self.min_stack.append(min(val, cur_min))

    def pop(self):
        self.stack.pop()
        self.min_stack.pop()

    def top(self): return self.stack[-1]
    def getMin(self): return self.min_stack[-1]
⌛ Complexity
Time: O(1) all ops Space: O(n)

Decode String
Problem (LeetCode 394): Given an encoded string like '3[a2[c]]', decode it to 'accaccacc'. Numbers can be multi-digit. Brackets can be nested arbitrarily deep.
Stack models nested scope: Maintain a stack of (current_count, string_built_so_far). When '[' is encountered, push state and reset. When ']' is encountered, pop and multiply. Current number and current string are tracked as running state.
🔍 Step-by-Step Trace
Input: "3[a2[c]]" curr_str="" curr_num=0 stack=[] '3': curr_num=3 '[': push ("", 3) to stack, reset: curr_str="", curr_num=0 stack=[("",3)] 'a': curr_str="a" '2': curr_num=2 '[': push ("a",2) to stack, reset: curr_str="", curr_num=0 stack=[("",3),("a",2)] 'c': curr_str="c" ']': pop ("a",2): curr_str = "a" + "c"*2 = "acc" stack=[("",3)] ']': pop ("",3): curr_str = "" + "acc"*3 = "accaccacc" stack=[] Result: "accaccacc" ✓
💻 Solution
Python · Decode String
def decode_string(s):
    stack = []
    curr_str = ""
    curr_num = 0

    for ch in s:
        if ch.isdigit():
            curr_num = curr_num * 10 + int(ch)  # handle multi-digit
        elif ch == '[':
            stack.append((curr_str, curr_num))  # save state
            curr_str, curr_num = "", 0
        elif ch == ']':
            prev_str, count = stack.pop()
            curr_str = prev_str + curr_str * count  # expand
        else:
            curr_str += ch
    return curr_str
⌛ Complexity
Time: O(n × max_repeat) Space: O(n)

Sliding Window Maximum
Problem (LeetCode 239): Given an array and integer k, return the maximum value in each sliding window of size k. E.g. [1,3,-1,-3,5,3,6,7] k=3 → [3,3,5,5,6,7].
Monotonic decreasing deque: Deque stores indices in decreasing order of their values. Invariant: front = index of current window's max. Before adding element i: remove from BACK all indices whose values are smaller (they can never be the future max). Remove from FRONT any index that's outside the window.
🔍 Step-by-Step Trace
nums=[1,3,-1,-3,5,3,6,7] k=3 i=0 v=1: deque=[0] i=1 v=3: 3>nums[0]=1 → pop 0, deque=[1] i=2 v=-1: -1<3, just append. deque=[1,2] Window [0,2] full → max=nums[deque[0]]=nums[1]=3 ✓ i=3 v=-3: append. deque=[1,2,3] Front=1 still in window [1,3] → max=nums[1]=3 ✓ i=4 v=5: 5>nums[3]=-3 → pop 3 5>nums[2]=-1 → pop 2 5>nums[1]=3 → pop 1 deque=[4] max=nums[4]=5 ✓ i=5 v=3: append. deque=[4,5]. max=nums[4]=5 ✓ i=6 v=6: 6>nums[5]=3→pop5, 6>nums[4]=5→pop4. deque=[6]. max=6 ✓ i=7 v=7: 7>nums[6]=6→pop6. deque=[7]. max=7 ✓ Result: [3,3,5,5,6,7] ✓
💻 Solution
Python · Sliding Window Maximum (Monotonic Deque)
from collections import deque
def max_sliding_window(nums, k):
    dq = deque()             # stores indices, decreasing by value
    result = []

    for i, n in enumerate(nums):
        # Remove indices outside window
        if dq and dq[0] < i - k + 1: dq.popleft()
        # Maintain decreasing invariant
        while dq and nums[dq[-1]] < n: dq.pop()
        dq.append(i)
        # Window is full
        if i >= k - 1:
            result.append(nums[dq[0]])  # front = max
    return result
⌛ Complexity
Time: O(n) Space: O(k)
