🧠 The Story — Google Maps
Real World Story — Navigation

Every time you ask Google Maps for directions, it runs a graph algorithm on a graph with 100M+ nodes.

Cities are vertices. Roads are edges. Travel time is the weight.

BFS finds the shortest path in an unweighted graph (fewest roads to cross).
Dijkstra finds the shortest path in a weighted graph (minimum total travel time).
DFS explores all paths (used for connectivity, cycle detection, topological sort).
Bellman-Ford handles negative weights (like credits that reduce travel time).

The graph has a direction (one-way streets) making it a directed graph (digraph). Most road networks are weighted digraphs — and every social network, dependency tree, and web is too.

Graph Representation: Adjacency List vs Matrix

Adjacency List: graph[u] = [(v, weight), ...]. Space: O(V+E). Best for sparse graphs.
Adjacency Matrix: matrix[u][v] = weight (0 if no edge). Space: O(V²). Best for dense graphs or when you need O(1) edge lookup.

In interviews: always use adjacency list unless the problem explicitly needs matrix representation.

🎬 BFS — Shortest Path in Unweighted Graph
🌊
BFS mental model: Ripples in a pond. Start from source. All distance-1 neighbors first. Then distance-2. Naturally finds shortest path.
When to use BFS: Unweighted shortest path. Level-order traversal. Minimum steps. "Connected in K moves?" problems.
Graph: 1-2, 1-3, 2-4, 3-4, 4-5. Find shortest path 1→5. Queue: [1] visited={1} Pop 1 → add 2,3 queue=[2,3] dist={1:0, 2:1, 3:1} Pop 2 → add 4 queue=[3,4] dist={4:2} Pop 3 → 4 already visited Pop 4 → add 5 queue=[5] dist={5:3} Pop 5 → DONE! Shortest path 1→5 = 3 steps ✓
Python · BFS — Shortest Path (Unweighted)
from collections import deque

def bfs_shortest_path(graph, src, dst):
    # graph: adjacency list {node: [neighbors]}
    queue = deque([(src, [src])])        # (current_node, path_so_far)
    visited = {src}

    while queue:
        node, path = queue.popleft()
        if node == dst: return path      # found! BFS guarantees shortest
        for nbr in graph[node]:
            if nbr not in visited:
                visited.add(nbr)
                queue.append((nbr, path + [nbr]))

    return []                            # no path exists

# For just the distance (more memory efficient):
def bfs_distance(graph, src, dst):
    queue = deque([(src, 0)])
    visited = {src}
    while queue:
        node, dist = queue.popleft()
        if node == dst: return dist
        for nbr in graph.get(node, []):
            if nbr not in visited:
                visited.add(nbr)
                queue.append((nbr, dist+1))
    return -1
🧠 Intuition — why BFS finds the shortest path for free

Imagine dropping a stone in a pond. The ripple reaches every point exactly in order of distance — distance-1 ring before distance-2, always. BFS is that ripple: the FIFO queue guarantees you finish processing all nodes at distance d before you ever touch a node at distance d+1. So the first time you reach the target, you reached it by the fewest edges — no shorter route can exist, because a shorter one would have surfaced in an earlier ring. That's why BFS needs no edge weights and no priority queue: the queue order is the distance order.

🌊 The ripple invariant

At any moment the queue holds nodes from at most two adjacent levels (d and d+1). Pop in, push out — distances only ever grow by one.

🟢 Mark visited on ENQUEUE

Mark a node the instant you add it to the queue, not when you pop it. Otherwise the same node gets queued by multiple neighbors and you revisit — turning O(V+E) into a blow-up.

🟠 When BFS is the tool

Unweighted shortest path, fewest-moves puzzles, level-order traversal, "reachable in K steps?", and multi-source flood fill (01-Matrix, rotting oranges).

🟣 Why not DFS here?

DFS dives down one branch first, so it may reach the target via a long detour before the short route. Only the breadth-first ring order guarantees minimality.

🌊 BFS ripples outward — first arrival = shortest path (1 → 5)
frontier (current ring) explored deeper target / shortest path
Each ring is one BFS level. Nodes light up strictly in distance order, so the first time node 5 is reached it is reached optimally. ▶ loops every 11s.
🎬 DFS — Connectivity, Cycles, and All Paths
Python · DFS — Iterative & Recursive Templates
# --- Recursive DFS ---
def dfs(graph, node, visited=None, path=None):
    if visited is None: visited = set(); path = []
    visited.add(node)
    path.append(node)
    for nbr in graph.get(node, []):
        if nbr not in visited:
            dfs(graph, nbr, visited, path)

# --- Cycle Detection in Directed Graph ---
# Use 3 states: WHITE(0)=unvisited, GRAY(1)=in-progress, BLACK(2)=done
def has_cycle_directed(graph, n):
    color = [0] * n                      # WHITE for all
    def dfs(u):
        color[u] = 1                     # GRAY: currently exploring
        for v in graph.get(u, []):
            if color[v] == 1: return True   # back edge → cycle!
            if color[v] == 0 and dfs(v): return True
        color[u] = 2                     # BLACK: done
        return False
    return any(dfs(u) for u in range(n) if color[u] == 0)

# --- Topological Sort (DFS postorder = reverse topo order) ---
def topo_sort(graph, n):
    visited, result = set(), []
    def dfs(u):
        visited.add(u)
        for v in graph.get(u, []):
            if v not in visited: dfs(v)
        result.append(u)                 # add AFTER visiting all descendants
    for u in range(n):
        if u not in visited: dfs(u)
    return result[::-1]                  # reverse postorder = topological order
🧠 Intuition — DFS and the three colors of cycle detection

DFS commits to a path and dives as deep as it can before backing up — like exploring a maze always taking the first unexplored door, retreating only at dead ends. The magic for directed cycle detection is realizing a node has three states, not two. WHITE = never seen. GRAY = on the current recursion path (entered, not yet finished). BLACK = fully explored and popped. If DFS ever points an edge at a GRAY node, that node is an ancestor still on the stack — you've looped back onto your own path. That's a back edge, and a back edge in a directed graph means a cycle. A BLACK neighbor is harmless: it's done, you just won't revisit it.

⚪ WHITE → 🟡 GRAY → ⚫ BLACK

GRAY on the way down (push), BLACK on the way up (pop). The GRAY set is exactly the current call stack.

🟢 BLACK ≠ cycle

Revisiting a finished (BLACK) node is fine — it just means two paths converge. Only a GRAY hit is a real cycle. Two-color visited sets can't tell these apart in directed graphs.

🟠 What DFS unlocks

Connected components, cycle detection, topological sort (reverse postorder), bridges/articulation points, and "all paths" enumeration.

🟣 Recursion = the stack

The implicit call stack is your path. Postorder (append on the way out) gives topo order because a node is finished only after all its descendants.

🔁 3-color DFS — a back edge to a GRAY node reveals a cycle
WHITE (unseen) GRAY (on path) BLACK (done) back edge → cycle
DFS turns nodes GRAY on entry and stacks them. When the edge C → A points at a still-GRAY ancestor, that's a back edge — proof of a directed cycle. ▶ loops every 11s.
🎬 Dijkstra's Algorithm — Weighted Shortest Path
🗺️
Story: GPS navigating a city. Roads have different travel times (weights). Greedily explore the currently-closest-unvisited node. Guarantee: when a node is finalized, its distance is optimal.
Why it works: We always expand the minimum distance node. Since all weights are non-negative, no future path can improve an already-finalized node's distance.
Graph: A→B(4), A→C(1), C→B(2), B→D(1). Find shortest from A. heap=[(0,A)] dist={A:0} Pop (0,A): explore B(4), C(1) → heap=[(1,C),(4,B)] dist={B:4,C:1} Pop (1,C): explore B(1+2=3) → update B! heap=[(3,B),(4,B)] dist={B:3} Pop (3,B): explore D(3+1=4) → heap=[(4,B),(4,D)] dist={D:4} Pop (4,B): already processed, skip Pop (4,D): DONE. Shortest: A→C→B→D = 4 ✓
Python · Dijkstra — O((V+E) log V)
import heapq

def dijkstra(graph, src, n):
    # graph[u] = [(weight, v), ...]  adjacency list
    dist = [float('inf')] * n
    dist[src] = 0
    heap = [(0, src)]           # (distance, node)

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]: continue   # stale entry: skip
        for w, v in graph[u]:
            nd = dist[u] + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(heap, (nd, v))

    return dist              # dist[i] = shortest distance from src to i

# ⚠️ Dijkstra FAILS with negative weights (use Bellman-Ford instead)
# The "stale entry" check (if d > dist[u]) is crucial for correctness.
# Without it, we'd re-process nodes and get wrong answers.
🧠 Intuition — Dijkstra is BFS that respects distance

BFS expands by number of edges because every edge costs the same. Once edges have different weights, "fewest edges" ≠ "shortest distance" — a 3-hop highway can beat a 1-hop dirt road. Dijkstra fixes this by replacing the plain queue with a min-heap keyed on distance: instead of expanding the nearest-in-hops node, it always expands the nearest-in-total-cost unfinished node. Because all weights are non-negative, the moment a node is popped with the smallest tentative distance, nothing can ever make it cheaper — every other route would have to pass through a node that's already at least as far. So that distance is final. Relaxation is the one operation: for each neighbor, check if going "through me" beats its current best, and if so, lower it and push the improved entry.

🔺 The greedy guarantee

Pop-min with non-negative weights ⇒ the popped node's distance is optimal. This breaks the instant a negative edge exists — then use Bellman-Ford.

🟢 Relaxation

if dist[u]+w < dist[v]: dist[v]=dist[u]+w; push. Lowering a neighbor's best-known distance is the only update the algorithm ever makes.

🟠 The stale-entry skip

We never delete old heap entries; we just push improved ones. When a node pops with d > dist[u], it's an outdated copy — continue past it. This keeps the heap simple and correct.

🟣 Cost

Each edge causes at most one push: O((V+E) log V) with a binary heap. The log V is the heap operation; lazy deletion avoids decrease-key.

🗺️ Dijkstra from A — greedy pops, relaxation lowers B from 4 → 3
finalized (optimal) relaxation / tentative edge weight
Pop the closest unfinished node, then relax its edges. Going A→C→B (1+2=3) beats the direct A→B (4), so B drops to 3; the old (4,B) heap entry is later skipped as stale. ▶ loops every 12s.
🎬 Bellman-Ford & Floyd-Warshall
Python · Bellman-Ford (handles negatives) & Floyd-Warshall (all-pairs)
# --- Bellman-Ford: Single Source, Handles Negative Weights ---
# O(V×E) time. Relax ALL edges V-1 times. Then check for negative cycle.
def bellman_ford(n, edges, src):
    # edges = [(u, v, weight), ...]
    dist = [float('inf')] * n
    dist[src] = 0

    for _ in range(n - 1):              # relax V-1 times
        for u, v, w in edges:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w

    # Check negative cycle: if any edge can still be relaxed → cycle
    for u, v, w in edges:
        if dist[u] + w < dist[v]:
            return None              # negative cycle detected!
    return dist

# --- Floyd-Warshall: All-Pairs Shortest Path --- O(V³) time, O(V²) space
def floyd_warshall(n, edges):
    INF = float('inf')
    dist = [[INF]*n for _ in range(n)]
    for i in range(n): dist[i][i] = 0
    for u, v, w in edges: dist[u][v] = w

    for k in range(n):                  # try routing through each intermediate k
        for i in range(n):
            for j in range(n):
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
    return dist

Number of Connected Components
Problem (LeetCode 323): You have n nodes (0 to n-1) and a list of undirected edges. Count the number of connected components.
DFS/BFS traversal: For each unvisited node, launch a DFS that marks all reachable nodes as visited. Each launch = 1 new component. Union-Find is an alternative but BFS/DFS is simpler to explain.
🔍 Step-by-Step Trace
n=5, edges=[[0,1],[1,2],[3,4]] Build adjacency list: 0: [1] 1: [0,2] 2: [1] 3: [4] 4: [3] visited = {F,F,F,F,F} Iteration: node 0: unvisited → DFS → visits {0,1,2} components=1 node 1: visited, skip node 2: visited, skip node 3: unvisited → DFS → visits {3,4} components=2 node 4: visited, skip Answer = 2 components ✓
💻 Solution
Python · Connected Components (DFS)
def count_components(n, edges):
    adj = [[] for _ in range(n)]
    for u, v in edges:
        adj[u].append(v); adj[v].append(u)
    visited = [False] * n

    def dfs(node):
        visited[node] = True
        for nb in adj[node]:
            if not visited[nb]: dfs(nb)

    count = 0
    for i in range(n):
        if not visited[i]:
            dfs(i); count += 1   # new component found
    return count

Time: O(V+E) Space: O(V+E)

Network Delay Time (Dijkstra)
Problem (LeetCode 743): n nodes, directed weighted edges. Send signal from node k. Find time for ALL nodes to receive signal. Return -1 if any unreachable.
Single-source shortest path — Dijkstra: BFS with a min-heap. dist[node] = minimum time signal takes to arrive. The last node to receive signal = max(dist). If any node has dist=∞, return -1.
🔍 Step-by-Step Trace
n=4, edges=[[2,1,1],[2,3,1],[3,4,1]], k=2 Build adj: 2→[(1,1),(3,1)] 3→[(4,1)] Dijkstra from node 2: heap: [(0,2)] dist={2:0} Pop (0,2): neighbors 1(cost=1), 3(cost=1) → dist[1]=1, dist[3]=1, push (1,1),(1,3) Pop (1,1): no outgoing edges Pop (1,3): neighbor 4(cost=1) → dist[4]=1+1=2, push (2,4) Pop (2,4): no outgoing edges dist = {2:0, 1:1, 3:1, 4:2} max(dist.values()) = 2 ✓
💻 Solution
Python · Network Delay Time — Dijkstra
import heapq
def network_delay_time(times, n, k):
    adj = {i:[] for i in range(1, n+1)}
    for u,v,w in times: adj[u].append((v,w))

    dist = {i: float('inf') for i in range(1, n+1)}
    dist[k] = 0
    heap = [(0, k)]                       # (cost, node)

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]: continue  # stale entry
        for v, w in adj[u]:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                heapq.heappush(heap, (dist[v], v))

    ans = max(dist.values())
    return ans if ans < float('inf') else -1
⌛ Complexity
Time: O((V+E) log V) Space: O(V+E)

Course Schedule — Cycle Detection & Topological Sort
Problem (LeetCode 207/210): numCourses courses, prerequisites pairs [a,b] mean 'take b before a'. Can you finish all? (207) / Return valid ordering (210). Essentially: detect cycle in directed graph, or find topological sort.
Kahn's BFS (Topological Sort): Build indegree array. Push all nodes with indegree=0 into queue. Process each: decrement neighbors' indegree; push neighbors reaching 0. If all nodes processed → no cycle. For ordering, collect nodes as processed.
🔍 Step-by-Step Trace
n=4, prereqs=[[1,0],[2,0],[3,1],[3,2]] Dependency graph: 0→1, 0→2, 1→3, 2→3 indegree: [0, 1, 1, 2] BFS: Queue: [0] (only node with indegree 0) Pop 0: order=[0], neighbors 1,2 → indegree=[0,0,0,2] Push 1,2. Queue: [1,2] Pop 1: order=[0,1], neighbor 3 → indegree=[0,0,0,1]. Queue: [2] Pop 2: order=[0,1,2], neighbor 3 → indegree=[0,0,0,0]. Push 3. Pop 3: order=[0,1,2,3] len(order)==4==n → No cycle. Valid ordering: [0,1,2,3] ✓
💻 Solution
Python · Course Schedule II (Kahn's BFS)
from collections import deque
def find_order(numCourses, prerequisites):
    indegree = [0] * numCourses
    adj = [[] for _ in range(numCourses)]
    for a, b in prerequisites:
        adj[b].append(a)              # b must come before a
        indegree[a] += 1

    q = deque([i for i in range(numCourses) if indegree[i]==0])
    order = []
    while q:
        node = q.popleft()
        order.append(node)
        for nb in adj[node]:
            indegree[nb] -= 1
            if indegree[nb] == 0: q.append(nb)
    return order if len(order)==numCourses else []

Time: O(V+E) Space: O(V+E)

01 Matrix — Multi-Source BFS
Problem (LeetCode 542): Given an m×n binary matrix, return a matrix where each cell contains its distance to the nearest 0.
Multi-source BFS (reverse thinking): Instead of BFS from each 1 to find nearest 0 (O(m²n²)), add ALL 0-cells to the queue at once and BFS outward. Each 1-cell gets updated the first time it's reached (shortest distance). This is O(mn).
🔍 Step-by-Step Trace
Matrix: [[0,0,0], [0,1,0], [1,1,1]] Init: Add all 0-cells to queue with dist=0 Queue: [(0,0),(0,1),(0,2),(1,0),(1,2)] dist matrix: [[0,0,0], [0,?,0], [?,?,?]] BFS step 1 (process all level-0 cells): From (1,0)→(2,0): dist=1 From (1,2)→(2,2): dist=1 dist[1][1]=1 (reached from (1,0) or (0,1)) BFS step 2: From (2,0)→(2,1): dist=2 From (2,2)→(2,1): min already set to 2 From (1,1)→(2,1): dist=2 (already found) Result: [[0,0,0], [0,1,0], [1,2,1]] ✓
💻 Solution
Python · 01 Matrix — Multi-Source BFS
from collections import deque
def update_matrix(mat):
    m, n = len(mat), len(mat[0])
    dist = [[float('inf')] * n for _ in range(m)]
    q = deque()

    for r in range(m):            # seed all 0-cells
        for c in range(n):
            if mat[r][c] == 0:
                dist[r][c] = 0; q.append((r,c))

    dirs = [(1,0),(-1,0),(0,1),(0,-1)]
    while q:
        r, c = q.popleft()
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < m and 0 <= nc < n and dist[nr][nc] > dist[r][c]+1:
                dist[nr][nc] = dist[r][c] + 1
                q.append((nr,nc))
    return dist
⌛ Complexity
Time: O(m×n) Space: O(m×n)
