🧠 The Story — The Treasure Hunt
The Algorithmic Mindset

An algorithm is a precise, finite sequence of steps to solve a problem. You use algorithms every day without realising it.

When you follow a recipe you're executing an algorithm. When you find parking you're running a search. When you rank search results you're running a scoring function.

The leap from "coding" to "algorithmic thinking" is learning to see a problem as: What are the inputs? What are the valid outputs? What is the constraint? Only then do you write code.

🔧 The 5-Step Design Process
  • Define the problem precisely. What is the input? What are valid outputs? Are there constraints (size, range, duplicates)?
  • Work a small example by hand. Pick n=4 or n=5, trace manually. This reveals patterns you'd miss just reading the spec.
  • Write brute force first. The simplest correct solution — no matter how slow. It is your reference implementation and your sanity check.
  • Identify the bottleneck. Profile or analyse: which loop is doing redundant work? Usually one nested loop hides the answer.
  • Optimise via patterns. Hash for O(1) lookup, sort to enable binary search, sliding window to avoid re-scanning, DP to avoid re-computing.
Example walk-through: "Find two numbers that sum to target" INPUT: nums = [2, 7, 11, 15], target = 9 OUTPUT: indices [0, 1] Step 1 – Define Input : int[] nums, int target Output: int[] of two indices i, j (i ≠ j, nums[i]+nums[j]==target) Constraint: exactly one solution guaranteed, 2 ≤ len ≤ 10⁴ Step 2 – Small example (by hand) n=4 → try all pairs: (0,1)=9 ✓ done. Step 3 – Brute force For every pair (i,j) with i<j, check if sum == target. Time O(n²), Space O(1) Step 4 – Bottleneck: the inner loop searches for complement. For nums[0]=2, we want to know: "Is 7 in the array?" Re-scanning array each time = O(n) per element = O(n²) total. Step 5 – Optimise: hash map gives O(1) lookup Pass through once. For each x, check if (target-x) already seen. Time O(n), Space O(n) ← best possible for unsorted input
Python · Two Sum — Brute Force → Optimal
# Brute force — O(n²)
def two_sum_brute(nums, target):
    for i in range(len(nums)):
        for j in range(i+1, len(nums)):
            if nums[i] + nums[j] == target:
                return [i, j]

# Optimal — O(n) using complement lookup
def two_sum(nums, target):
    seen = {}                          # val → index
    for i, x in enumerate(nums):
        complement = target - x
        if complement in seen:
            return [seen[complement], i]
        seen[x] = i
🔎 Pattern Recognition — The Cheat Sheet

When you see a problem, scan for these signals to immediately know the algorithmic family:

🔍 "Find if exists"
HashSet — O(1) lookup. Sort + binary search if sorted.
📈 "Longest / Shortest subarray"
Sliding window (unsorted) or two pointers.
⚡ "All combinations / subsets"
Backtracking with pruning.
📄 "Max/min over choices"
Dynamic programming — overlapping subproblems.
🌎 "Connected groups"
BFS/DFS on graph or matrix.
🔄 "Sorted + search"
Binary search — always O(log n).
📊 "Rank / k-th element"
Heap or QuickSelect.
🆕 "Parentheses / nested"
Stack — LIFO structure mirrors nesting.

🎯 Interview Questions — Detailed Walkthroughs
1
Analyse Two Nested Loops
Problem: What is the time complexity of this code? for i in range(n): for j in range(i+1,n): if arr[i]>arr[j]: arr[i],arr[j]=arr[j],arr[i]
Count operations: The inner loop runs (n-1) + (n-2) + ... + 1 = n(n-1)/2 times. Drop constants and lower-order terms → O(n²). This is Bubble Sort's comparison count.
🔍 Step-by-Step Trace
Outer i=0: inner runs n-1 times (j=1,2,...,n-1) Outer i=1: inner runs n-2 times (j=2,3,...,n-1) Outer i=2: inner runs n-3 times ... Outer i=n-2: inner runs 1 time Total = (n-1) + (n-2) + ... + 1 = n(n-1) / 2 = (n² - n) / 2 ≈ n²/2 Drop constant factor 1/2 → O(n²) Rule: when inner loop depends on outer variable and runs roughly proportionally, result is O(n²).
💻 Solution
Python · Count comparisons explicitly
def count_ops(n):
    ops = 0
    for i in range(n):
        for j in range(i+1, n):
            ops += 1
    return ops

for n in [10, 100, 1000]:
    print(f"n={n}: ops={count_ops(n)}, formula={n*(n-1)//2}")
# n=10: ops=45, formula=45 ✓
⌛ Complexity
Time: O(n²) Space: O(1)
2
Brute Force → Optimal: Contains Duplicate
Problem (LeetCode 217): Given array, return True if any value appears at least twice. E.g. [1,2,3,1] → True.
Three approaches: (1) Brute force O(n²) — compare every pair. (2) Sort O(n log n) — check adjacent elements. (3) HashSet O(n) — O(1) lookup. Always start at brute force and optimise systematically.
🔍 Step-by-Step Trace
Approach 1 — Brute Force O(n²) For every pair (i,j), check if arr[i]==arr[j] 4 elements → 6 comparisons [1,2,3,1]: (0,3): 1==1 → True ✓ Approach 2 — Sort + scan O(n log n) Sort: [1,1,2,3] Scan adjacent: arr[0]==arr[1]? 1==1 → True ✓ Space O(1) if sort in-place Approach 3 — HashSet O(n) seen = {} x=1: not in seen → add x=2: not in seen → add x=3: not in seen → add x=1: IN seen → return True ✓ Interview tip: mention all 3, then justify choosing HashSet for O(n) time. If memory is constrained, offer sort approach.
💻 Solution
Python · Contains Duplicate (all 3 approaches)
# O(n) — HashSet
def contains_duplicate(nums):
    seen = set()
    for x in nums:
        if x in seen: return True
        seen.add(x)
    return False

# One-liner (same complexity)
def contains_dup2(nums):
    return len(nums) != len(set(nums))
⌛ Complexity
Time: O(n) Space: O(n)
3
Identify the Pattern: Sliding Window vs Two Pointers
Question: You have an unsorted array and must find the max sum of any contiguous subarray of length k. Which technique applies? What if k is variable and you want the smallest subarray with sum ≥ target?
Fixed window → sliding window. Maintain a running sum of size k — add right element, remove left element each step. Variable window → two pointers: expand right to meet target, shrink left to minimise, track minimum width.
🔍 Step-by-Step Trace
FIXED WINDOW (k=3): [2,1,5,1,3,2], k=3 Window [2,1,5] = 8 Slide → remove 2, add 1: [1,5,1] = 7 Slide → remove 1, add 3: [5,1,3] = 9 ← max Slide → remove 5, add 2: [1,3,2] = 6 Answer: 9 VARIABLE WINDOW: find min length with sum≥7 [2,3,1,2,4,3], target=7 lo=0 hi=0 sum=2 < 7 → expand right lo=0 hi=3 sum=8 ≥ 7 → shrink left (size=4) lo=1 hi=3 sum=6 < 7 → expand right lo=1 hi=4 sum=10 ≥ 7 → shrink left (size=4) lo=2 hi=4 sum=7 ≥ 7 → shrink left (size=3) lo=3 hi=4 sum=6 < 7 → expand right lo=3 hi=5 sum=9 ≥ 7 → shrink left (size=3) lo=4 hi=5 sum=7 ≥ 7 → shrink left (size=2) ← min! Answer: 2
💻 Solution
Python · Max Fixed Window + Min Variable Window
def max_fixed_window(arr, k):
    window_sum = sum(arr[:k])
    best = window_sum
    for i in range(k, len(arr)):
        window_sum += arr[i] - arr[i-k]
        best = max(best, window_sum)
    return best

def min_subarray_len(target, nums):
    lo, cur, best = 0, 0, float('inf')
    for hi, x in enumerate(nums):
        cur += x
        while cur >= target:
            best = min(best, hi - lo + 1)
            cur -= nums[lo]; lo += 1
    return best if best != float('inf') else 0
⌛ Complexity
Time: O(n) Space: O(1)
