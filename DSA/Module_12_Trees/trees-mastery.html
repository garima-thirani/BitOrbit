🧠 The Story — The Family Tree
Real World Story — Family Genealogy

A family tree: you, your parents, your grandparents, your children.

Every person (node) has a parent (except the great-grandparent = root) and up to two children (left, right in a binary tree).

When you ask "how many descendants does person X have?", you think recursively: count X's left subtree descendants + right subtree descendants + 1 (for X).

This is the universal tree recursion template:
solve(node) = combine(solve(node.left), solve(node.right), node.val)

Height, size, diameter, max path sum, LCA — they ALL follow this pattern. Master the template, master trees.

The Universal Tree Recursion Template
def solve(node):
  if node is None: return BASE_CASE
  left = solve(node.left) # solve left subtree
  right = solve(node.right) # solve right subtree
  return combine(left, right, node.val) # merge results

Base case: what to return for an empty tree (often 0, [], False, None, or infinity). Combine: depends on the problem.

💡 The Deep Intuition — One Template Solves Almost Everything

Here is the mental unlock: a tree is a self-similar object. The left subtree is itself a smaller tree; so is the right. So any question you can ask about the whole tree, you can ask about each subtree — and the answer for the whole is just a tiny bit of arithmetic on the two sub-answers. You never "loop over a tree." You delegate to the children, trust the answer comes back correct, and combine. That act of trust is the leap of faith of recursion.

🟡 How to recognize it

Reach for tree recursion whenever the question is about a node and everything beneath it: "height", "does a path exist", "sum of a subtree", "is it balanced", "deepest common ancestor". If you can phrase the answer as "my answer = f(left's answer, right's answer, me)", you have it.

🔵 What flows where

Two directions of information. Down (passed as arguments): context a node needs from its ancestors — like valid value bounds, current depth, or the path so far. Up (the return value): the summarised answer a node hands back to its parent — like height, sum, or a found node.

🟣 The recurrence

The single line every tree problem specialises:

solve(node) = combine(
  solve(node.left),
  solve(node.right),
  node.val)

🟢 The base case

The empty tree (node is None) is the floor of the recursion. Its answer is the identity for your combine step: 0 for height/sum, True for "all valid", None for search, −∞ for max-path. Pick the value that makes the parent's arithmetic come out right.

🌍 Where trees actually live around you
  • Your file system: folders contain folders contain files. "How big is this folder?" = sum of every child's size — the exact subtree-sum recurrence.
  • A web page (the DOM): every <div> holds nested elements. The browser computes layout by recursing the tree, bottom-up, just like computing height.
  • A company org chart: "who is the lowest manager over both Alice and Bob?" is literally the Lowest Common Ancestor problem.
  • Comment threads / family trees: replies under replies, descendants of descendants — counting them is a subtree count.
  • A tournament bracket: the winner bubbles up from the leaves — the same upward flow as height or max-path-sum.
⚠️ The trap that bites everyone

Never write def f(node, result=[]) with a mutable default. Python creates that list once, so a second call sees the first call's leftovers. Pass the accumulator explicitly, use a closure, or default to None and create it inside. Second trap: confusing height (edges going down to the deepest leaf) with depth (edges going up to the root) — they are mirror images.

🎬 All Four Traversals — DFS & BFS
Python · All Traversals
# --- Pre-order: root, left, right (good for tree copy/serialization) ---
def preorder(node, result=[]):
    if not node: return
    result.append(node.val)           # root FIRST
    preorder(node.left, result)
    preorder(node.right, result)

# --- In-order: left, root, right (BST → sorted output) ---
def inorder(node, result=[]):
    if not node: return
    inorder(node.left, result)
    result.append(node.val)           # root BETWEEN left and right
    inorder(node.right, result)

# --- Post-order: left, right, root (good for deletion, size computation) ---
def postorder(node, result=[]):
    if not node: return
    postorder(node.left, result)
    postorder(node.right, result)
    result.append(node.val)           # root LAST

# --- Level-order (BFS): level by level ---
from collections import deque
def level_order(root):
    if not root: return []
    result, queue = [], deque([root])
    while queue:
        level = []
        for _ in range(len(queue)):     # process exactly one level
            node = queue.popleft()
            level.append(node.val)
            if node.left:  queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)
    return result
🎬 DFS in-order — dive left, emit the node, then dive right
In-order = Left · Root · Right visits 4·2·5·1·3. Notice the node is emitted only after its whole left subtree — that is why in-order on a BST comes out sorted. Pre-order would be 1·2·4·5·3 (root first); post-order 4·5·2·3·1 (root last). ▶ loops every 9s.
🎬 BFS level-order — a ripple expanding outward from the root
BFS uses a queue, not the call stack. By processing exactly len(queue) nodes per round, each round is one full level. This is why BFS finds the shortest path in an unweighted graph — it reaches every node at its minimum depth first. ▶ loops every 9s.
🎬 Height, Diameter, and Max Path Sum
Python · Height, Diameter, Max Path Sum
# --- Height of tree ---
def height(node):
    if not node: return 0
    return 1 + max(height(node.left), height(node.right))

# --- Diameter: longest path (may not go through root) ---
def diameter(root):
    best = [0]
    def height(node):
        if not node: return 0
        l = height(node.left)
        r = height(node.right)
        best[0] = max(best[0], l + r)    # diameter through this node
        return 1 + max(l, r)             # height returned to parent
    height(root)
    return best[0]

# --- Max Path Sum (can be any node to any node) ---
def max_path_sum(root):
    best = [float('-inf')]
    def gain(node):
        if not node: return 0
        l = max(gain(node.left),  0)    # discard negative paths
        r = max(gain(node.right), 0)
        best[0] = max(best[0], l + r + node.val)   # path through this node
        return node.val + max(l, r)                # max one-side gain to parent
    gain(root)
    return best[0]
🎬 Diameter — heights bubble UP from the leaves, the answer forms at each node
The trick: each node returns its height to its parent, but along the way it also checks L + R — the longest path that bends at this node — against a running best. One post-order pass, O(n). The diameter need not pass through the root; here it bends at node 1. ▶ loops every 11s.
height returned upward L+R candidate at a node winning diameter path
🎬 Lowest Common Ancestor (LCA)
💡
LCA story: In a company hierarchy, the lowest common manager of two employees is the deepest manager who is an ancestor of both.
Key insight: DFS post-order. If both p and q are found in different subtrees of node X, then X is the LCA.
Tree: 3 / 5 1 / \ / 6 2 0 8 / 7 4 LCA(5,1)=3 (they're in different subtrees of 3) LCA(5,4)=5 (4 is in 5's subtree, 5 itself is found) LCA(6,4)=5 (6 left of 5, 4 right of 5 → meet at 5)
Python · Lowest Common Ancestor
def lca(root, p, q):
    if not root: return None     # not found
    if root == p or root == q: return root   # found one of them

    left  = lca(root.left,  p, q)
    right = lca(root.right, p, q)

    if left and right: return root   # p and q in different subtrees: THIS IS LCA
    return left or right             # one side has both, return that result

# This handles: LCA when one node is ancestor of the other.
# If left is None but right is non-None → both p,q in right subtree → right holds LCA.
🎬 LCA(4,5) — each found node reports up; the first node hearing from BOTH sides is the answer
The "both sides non-null" rule. A node returns itself if it is p or q, otherwise it returns whatever its children found. The moment a node gets a non-null answer from both children, the two targets must live in separate subtrees — so this node is their lowest common ancestor. Above it, that answer just rides the call stack to the root. ▶ loops every 10s.

Symmetric Tree
Problem (LeetCode 101): Given the root of a binary tree, check whether it is a mirror of itself (symmetric around its center).
Mirror recursion: Two subtrees are mirrors if: (1) both are null, or (2) both are non-null AND their root values match AND left.left mirrors right.right AND left.right mirrors right.left. Notice: NOT left mirroring itself, but left mirroring right.
🔍 Step-by-Step Trace
Tree: 1 / 2 2 / \ / 3 4 4 3 isMirror(node2_left, node2_right): 2.val == 2.val ✓ isMirror(3, 3): 3==3, isMirror(null,null)✓, isMirror(null,null)✓ → True isMirror(4, 4): 4==4, similar → True → True Asymmetric example: Tree: 1 / 2 2 \ 3 3 isMirror(node2_L, node2_R): 2.val==2.val ✓ isMirror(null, 3): one null, one not → False ✗
💻 Solution
Python · Symmetric Tree
def is_symmetric(root):
    def mirror(left, right):
        if not left and not right: return True
        if not left or not right: return False   # one null
        return (left.val == right.val and
                mirror(left.left,  right.right) and   # outer pair
                mirror(left.right, right.left))           # inner pair
    return mirror(root.left, root.right) if root else True
⌛ Complexity
Time: O(n) Space: O(h) recursion stack

Path Sum II — All Root-to-Leaf Paths
Problem (LeetCode 113): Find all root-to-leaf paths in a binary tree that sum to a given target. Return all such paths.
DFS + Backtracking: Carry current path and remaining sum. At leaf: if remaining==0, record path. CRITICAL: append to path before recursing, pop after — this 'undo' is the essence of backtracking.
🔍 Step-by-Step Trace
Tree: 5 targetSum=22 / 4 8 / / 11 13 4 / \ 7 2 1 DFS trace (showing path accumulation): Visit 5: path=[5], rem=17 Visit 4: path=[5,4], rem=13 Visit 11: path=[5,4,11], rem=2 Visit 7: leaf, rem=2-7=-5 ≠ 0 ✗ BACKTRACK (pop 7) Visit 2: leaf, rem=2-2=0 ✓ → record [5,4,11,2] BACKTRACK (pop 2) BACKTRACK (pop 11) BACKTRACK (pop 4) Visit 8: path=[5,8], rem=9 Visit 13: leaf, rem=9-13=-4 ✗ Visit 4: path=[5,8,4], rem=5 Visit 1: leaf, rem=5-1=4 ≠ 0 ✗ BACKTRACK... Result: [[5,4,11,2]] ✓
💻 Solution
Python · Path Sum II
def path_sum(root, targetSum):
    result = []
    def dfs(node, remaining, path):
        if not node: return
        path.append(node.val)
        if not node.left and not node.right:  # leaf check
            if remaining == node.val:
                result.append(path[:])              # copy! not reference
        else:
            dfs(node.left,  remaining - node.val, path)
            dfs(node.right, remaining - node.val, path)
        path.pop()                                  # BACKTRACK
    dfs(root, targetSum, [])
    return result
⌛ Complexity
Time: O(n·h) for copying paths Space: O(h) stack + O(paths)

Serialize & Deserialize Binary Tree
Problem (LeetCode 297): Design an algorithm to serialize a binary tree to a string, and deserialize it back. Must be lossless — null pointers included.
Pre-order DFS with null markers: Serialize visits root→left→right, writing 'N' for None. Deserialize uses a global iterator/deque. Each recursive call consumes one token — the structure is implicitly encoded.
🔍 Step-by-Step Trace
Tree: 1 / 2 3 / 4 5 Serialize (pre-order, 'N'=null): visit 1: "1," visit 2: "2," left=null: "N," right=null: "N," visit 3: "3," visit 4: "4," left=null: "N," right=null: "N," visit 5: "5," left=null: "N," right=null: "N," → "1,2,N,N,3,4,N,N,5,N,N" Deserialize "1,2,N,N,3,4,N,N,5,N,N": tokens = deque([1,2,N,N,3,4,N,N,5,N,N]) pop 1: root=1 pop 2: left=2 pop N: left.left=None pop N: left.right=None pop 3: right=3 pop 4: right.left=4 pop N: ... pop N: ... pop 5: right.right=5 ... → Exact original tree ✓
💻 Solution
Python · Serialize & Deserialize Tree
from collections import deque
class Codec:
    def serialize(self, root):
        if not root: return 'N'
        return (str(root.val) + ',' +
                self.serialize(root.left) + ',' +
                self.serialize(root.right))

    def deserialize(self, data):
        tokens = deque(data.split(','))
        def build():
            val = tokens.popleft()
            if val == 'N': return None
            node = TreeNode(int(val))
            node.left  = build()          # consume left subtree
            node.right = build()          # consume right subtree
            return node
        return build()
⌛ Complexity
Time: O(n) Space: O(n)

Binary Tree Right Side View
Problem (LeetCode 199): Imagine looking at a binary tree from the right side. Return the values of the nodes you can see (one per level — the rightmost node at each depth).
BFS (level-order): Process level by level; take the last node in each level batch. DFS trick: Pre-order, but visit right child before left. Append to result only when depth == len(result) (first time visiting this depth = rightmost node).
🔍 Step-by-Step Trace
Tree: 1 / 2 3 \ 5 4 BFS approach: Level 0: [1] → rightmost = 1 Level 1: [2, 3] → rightmost = 3 Level 2: [5, 4] → rightmost = 4 Result: [1, 3, 4] ✓ DFS (right-first pre-order): Depth 0, visit 1: result=[], depth==len(result)? 0==0 ✓ → result=[1] Depth 1, visit 3: depth==len(result)? 1==1 ✓ → result=[1,3] Depth 2, visit 4: 2==2 ✓ → result=[1,3,4] Depth 2, visit 2: 2!=3 ✗ (already have depth 2) Depth 3, visit 5: 3==3 ✓ → result=[1,3,4,5]
💻 Solution
Python · Right Side View (BFS)
from collections import deque
def right_side_view(root):
    if not root: return []
    result, q = [], deque([root])
    while q:
        level_size = len(q)
        for i in range(level_size):
            node = q.popleft()
            if i == level_size - 1:
                result.append(node.val)  # last in this level = rightmost
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
    return result
⌛ Complexity
Time: O(n) Space: O(w) where w=max width
