🧠 The Story — The Hotel Registration Desk
Real World Story — Check-in System

A hotel with 10,000 rooms. Guest "Smith" checks in. You need to find their room instantly.

The hotel uses a key card system. Guest name → hash function → room number. O(1) lookup regardless of how many guests.

Hash Function: Maps a key (name) to an index (room). Good hash functions distribute evenly. Collisions (two names hashing to same room) are resolved with chaining (list at each slot) or open addressing (next free slot).

Python's dict and set are hash tables. Every time you do x in my_dict, Python computes hash(x) and does a table lookup in O(1). That's why the pattern "create a freq map, then query it" turns O(n^2) into O(n).

When to Reach for a Hash Map
  • "Two Sum" pattern: For each element, look up (target - element) in a hash map. O(n) instead of O(n^2).
  • Frequency counting: Count occurrences. Group anagrams, top K frequent, etc.
  • Seen before: Track visited states, deduplicate elements.
  • O(1) lookup for prefix info: Prefix sums combined with hash map for subarray problems.

A hash map is a time machine for lookups. Instead of scanning the array again to answer a question, you record the answer as you go so any future element can retrieve it in O(1). Two Sum doesn't search for a complement — it checks whether the complement was already seen. Subarray-sum-equals-k doesn't scan all sub-windows — it asks "have I seen a prefix sum that would make the current window work?" The trick is always the same: store what you know, look up what you need.

🔍 Two Sum / complement lookup

For each element x, ask: does target − x exist in the map? If yes, done. If no, record x. One pass, O(n).

∑ Prefix sum + frequency map

Running prefix s. We want s − prev_s = k, so look up s − k. Record each prefix in a frequency map as you produce it.

📆 LRU cache

HashMap gives O(1) key lookup; doubly-linked list gives O(1) "move to front" and "evict tail". Together: O(1) get and put. Python's OrderedDict is the same idea built-in.

📊 Frequency / grouping

Sort key (e.g. sorted(word)) maps anagrams to the same bucket. Character frequency tuple maps anagrams to the same key. Group by any canonical form.

two_sum: for x: if (target-x) in map → found. else map[x]=i
subarray_sum_k: freq={0:1}; prefix+=num; count+=freq[prefix-k]; freq[prefix]++

🌍 Where this lives in the real world
  • Database indexes: a hash index answers exact-match queries in O(1) — the engine's in-memory hash map for primary keys.
  • DNS cache: hostname → IP is a hash map; LRU eviction drops least-recently-queried entries when the cache fills.
  • Browser cache / CDN: URL hashed to a bucket; LRU eviction policy to stay within memory limits.
  • Anagram detection / spell checker: sort + group — or character frequency tuple — to cluster words that are rearrangements.
  • Sliding window analytics: "how many sessions in the last hour had at least k events?" maps directly to subarray-sum-k over a time series.
  • Rabin-Karp string search: hash the pattern; slide a rolling hash over the text — O(n+m) average instead of O(n·m).
⚠️ The traps

Initialise prefix freq with {0:1}: forgetting this misses subarrays that start at index 0, because you'd never record that a zero-length prefix has sum 0. Hash collisions: Python dicts don't collide, but custom hash functions in other languages do — use a good hash. LRU with OrderedDict: move_to_end on access, popitem(last=False) on eviction — if you forget to move on put of an existing key, you update the value but leave it in the wrong position.

Python · Hash Map and Counter — Essential Patterns
from collections import defaultdict, Counter

# --- Two Sum: O(n) with hash map ---
def two_sum(nums, target):
    seen = {}                              # val → index
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []

# --- Frequency Map: Counter ---
freq = Counter("leetcode")               # Counter({'e': 3, 'l': 1, ...})
most_common = freq.most_common(3)        # top 3 most frequent chars

# --- Group Anagrams: sort key ---
def group_anagrams(strs):
    groups = defaultdict(list)
    for word in strs:
        key = tuple(sorted(word))        # anagrams share same sorted key
        groups[key].append(word)
    return list(groups.values())

# --- Subarray Sum Equals K (prefix sum + hash map) ---
def subarray_sum(nums, k):
    count, prefix = 0, 0
    freq = {0: 1}                        # empty subarray has sum 0
    for num in nums:
        prefix += num
        count += freq.get(prefix - k, 0)
        freq[prefix] = freq.get(prefix, 0) + 1
    return count
🎬 Two Sum — look up the complement you already recorded
Store first, search later — the hash map answers any future "have you seen X?" in O(1). The complement check is a question about the past; you answer it by consulting the map you've already built, not by scanning back through the array. ▶ loops every 11s.

🎬 Subarray Sum = K — The Hash+Prefix Pattern
💡
The genius insight: If prefix[j] - prefix[i] = k, then subarray (i,j] sums to k. We want count of pairs (i,j) where prefix[j] - k = prefix[i]. For each j, look up prefix[j]-k in the frequency map of past prefix sums.
nums=[1,1,1], k=2 prefix=0, freq={0:1} num=1: prefix=1. look for 1-2=-1 → 0. freq={0:1,1:1} num=1: prefix=2. look for 2-2=0 → 1. count=1. num=1: prefix=3. look for 3-2=1 → 1. count=2. Result=2 (subarrays [1,1] at index 0 and 1) ✓
🎬 Subarray sum = k — track prefix sums, look up the gap you need
Prefix subtraction collapses any subarray into a constant-time question. The map keeps a running tally of every prefix sum seen. When you arrive at prefix s and ask "how many old prefixes equal s−k?", each match is a window from there to here that sums to exactly k. ▶ loops every 12s.
window [0,1] = 2 window [1,2] = 2
🎬 LRU Cache — Hash Map + Doubly Linked List
Python · LRU Cache — O(1) for both get and put
class LRUNode:
    def __init__(self, k=0, v=0):
        self.k = k; self.v = v; self.prev = self.nxt = None

class LRUCache:
    def __init__(self, cap):
        self.cap = cap
        self.cache = {}                        # key to node mapping
        self.head = LRUNode()                  # dummy head (most recent)
        self.tail = LRUNode()                  # dummy tail (least recent)
        self.head.nxt = self.tail; self.tail.prev = self.head

    def _remove(self, n):
        n.prev.nxt = n.nxt; n.nxt.prev = n.prev

    def _insert_front(self, n):
        n.nxt = self.head.nxt; n.prev = self.head
        self.head.nxt.prev = n; self.head.nxt = n

    def get(self, key):
        if key not in self.cache: return -1
        self._remove(self.cache[key]); self._insert_front(self.cache[key])
        return self.cache[key].v

    def put(self, key, val):
        if key in self.cache: self._remove(self.cache[key])
        self.cache[key] = LRUNode(key, val); self._insert_front(self.cache[key])
        if len(self.cache) > self.cap:
            lru = self.tail.prev
            self._remove(lru); del self.cache[lru.k]
🎬 LRU Cache — every get/put moves a node to the front
Two data structures, one contract. The hash map finds a node in O(1); the doubly-linked list removes and re-inserts it at the front in O(1). Without the list, HashMap alone can't answer "which key was least recently used" efficiently. Without the map, the list alone can't find a key in O(1). Together they're perfect. ▶ loops every 13s.
newly inserted (MRU) moved to front on get evicted (LRU)

Longest Consecutive Sequence — O(n)
Problem (LeetCode 128): Given an unsorted array of integers, find the length of the longest consecutive elements sequence. Must run in O(n) time.
Hash Set + smart start detection: Add all numbers to a set. Only START counting a streak when num-1 is NOT in the set (meaning this is a sequence head). Extend while num+1, num+2, ... exist in set. This ensures each number is visited at most twice.
🔍 Step-by-Step Trace
nums = [100,4,200,1,3,2] Set = {100,4,200,1,3,2} num=100: 99 not in set → START. Streak: 100,101? No. Length=1 num=4: 3 in set → skip (not start) num=200: 199 not in set → START. Length=1 num=1: 0 not in set → START. 2∈set: 3∈set: 4∈set: 5∉set → Length=4 num=3: 2 in set → skip num=2: 1 in set → skip Max streak: 4 → [1,2,3,4] ✓ Why O(n): Each number is a START at most once. The while loop across ALL start points visits each number at most once total.
💻 Solution
Python · Longest Consecutive Sequence
def longest_consecutive(nums):
    num_set = set(nums)
    best = 0
    for num in num_set:
        if num - 1 not in num_set:     # sequence start
            streak = 1
            curr = num + 1
            while curr in num_set:
                streak += 1
                curr += 1
            best = max(best, streak)
    return best
⌛ Complexity
Time: O(n) Space: O(n)

Top K Frequent Elements
Problem (LeetCode 347): Given an integer array, return the k most frequent elements. You may assume k is always valid. Aim for O(n) solution.
Bucket Sort on frequency: Count frequencies with a hash map. Create 'buckets' where buckets[f] = list of all numbers that appear exactly f times. Frequency is at most n, so scan buckets from high to low. O(n) vs O(n log k) for heap.
🔍 Step-by-Step Trace
nums = [1,1,1,2,2,3] k=2 Frequencies: {1:3, 2:2, 3:1} Buckets (index = frequency): buckets[1] = [3] buckets[2] = [2] buckets[3] = [1] Scan from high to low: i=6: empty ... i=3: [1] → result=[1], need 1 more i=2: [2] → result=[1,2], need 0 more Result = [1,2] ✓
💻 Solution
Python · Top K Frequent Elements (Bucket Sort)
def top_k_frequent(nums, k):
    count = {}
    for n in nums: count[n] = count.get(n, 0) + 1

    buckets = [[] for _ in range(len(nums) + 1)]
    for num, freq in count.items():
        buckets[freq].append(num)          # bucket by frequency

    result = []
    for freq in range(len(buckets)-1, 0, -1):  # high freq first
        result.extend(buckets[freq])
        if len(result) >= k: break
    return result[:k]
⌛ Complexity
Time: O(n) Space: O(n)

4Sum II
Problem (LeetCode 454): Given four integer arrays A, B, C, D (each length n), count how many tuples (i,j,k,l) such that A[i]+B[j]+C[k]+D[l] == 0.
Meet-in-the-middle hashing: Compute all A[i]+B[j] sums and store their counts in a hash map. Then for every C[k]+D[l] pair, check if -(C[k]+D[l]) exists in the map. This reduces O(n^4) to O(n^2).
🔍 Step-by-Step Trace
A=[1,2] B=[-2,-1] C=[-1,2] D=[0,2] Phase 1 — build sum_map: A[0]+B[0]=1+(-2)=-1 → count: {-1:1} A[0]+B[1]=1+(-1)=0 → count: {-1:1, 0:1} A[1]+B[0]=2+(-2)=0 → count: {-1:1, 0:2} A[1]+B[1]=2+(-1)=1 → count: {-1:1, 0:2, 1:1} Phase 2 — query with -(C+D): C[0]+D[0]=-1+0=-1 target= 1 → sum_map[1]=1 count+=1 C[0]+D[1]=-1+2=1 target=-1 → sum_map[-1]=1 count+=1 C[1]+D[0]=2+0=2 target=-2 → not in map C[1]+D[1]=2+2=4 target=-4 → not in map Total = 2 ✓
💻 Solution
Python · 4Sum II
def four_sum_count(A, B, C, D):
    sum_map = {}
    for a in A:
        for b in B:
            sum_map[a+b] = sum_map.get(a+b, 0) + 1

    count = 0
    for c in C:
        for d in D:
            count += sum_map.get(-(c+d), 0)  # look for complement
    return count
⌛ Complexity
Time: O(n²) Space: O(n²)
